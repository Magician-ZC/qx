## 一句话结论

当前项目已经不是空壳：**服务端权威战棋、LLM 单位自治、命运收件箱、大世界调度地基、PvE 威胁、跨玩家世界总线、成本/安全/鉴权硬化、商业化/合规最小垂直切片**都已经落到代码，并且后端 `go build` / `go test` / `go vet` 通过。

但它还不是一个完整可发行游戏。现在更准确的阶段是：

**可运行、可验证、已有核心技术闭环的 First Playable 原型；前台乐趣闭环正在接主界面，商业化/合规已有 flag-gated 纵切，但真实支付、公开发行和内容生产管线还没完成。**

## 已完成的部分

### 1. 传统战棋对局骨架

已完成：

- 服务端权威的单局状态机：部署阶段 / 执行阶段交替推进。
- 玩家通过自然语言下达 doctrine / task / order，单位具体行动由 LLM 或规则 fallback 决定。
- 六边形战场、地形、天气、视野、迷雾、单位状态、行动点、技能、战斗结算。
- 单人模式与双人房间基础流程。
- 前端 React + Pixi.js 战棋客户端，能渲染地图、单位、HUD、指令、日志与 WebSocket 实时更新。

判断：**战棋底盘可用，但 UI 仍偏工程原型。**

### 2. LLM 决策基础设施

已完成：

- 所有 LLM 调用统一走 `ai.Service.GenerateJSON()`。
- JSON Schema 强校验，不接受半截结构化结果。
- 多 provider fallback：OpenAI / DeepSeek / 后备端点。
- LLM 交互留痕、token 估算、成本估算、预算护栏。
- prompt cache 已实现，命中时标记 `CacheHit`，避免把缓存复用误记为真实成本。
- 反射层短路：日常安静 tick 可跳过 LLM，降低成本。
- 内容安全审核 chokepoint 已接入玩家输入和 AI 输出；当前由 `QUNXIANG_CONTENT_SAFETY` 控制，默认关闭。

判断：**LLM 工程地基比较扎实，已经有降本和失败兜底意识。**

### 3. 确定性结算原语

已完成：

- `engine/decision`：三层决策路由、归因校验、戏剧性动作前因门控。
- `engine/arbitration`：零和仲裁，胜率按 Score，避免行动频率和付费影响胜负。
- `engine/relevance`：相关性评分、传播衰减、命运路由、同意档位、撮合评分。
- `engine/encounter`：PvE 贡献、分赃、失败惩罚分级、单人解锁门。
- `engine/status`：状态变更统一经 Mutator，批量写入与乐观并发路径已具备。
- `engine/narration` / `attachment`：祖魂语气卡片与牵挂计算原语。

判断：**底层“决策用 LLM、结算用代码”的方向已经立住了。**

### 4. 命运开盒 MVP

已完成：

- 独立 Fate 前端入口：`#fate` 路由下的捏人 / 预览 / 四槽界面。
- 四槽界面：状态卡、高光卡、待决策、回响带。
- 命运事件路由：自治 / 高光 / 待决策。
- FateScore 已接入，不再只靠裸 relevance。
- 待决策处理已从“只打标记”升级为真实后果：通过 Mutator 改 morale / loyalty，并写标准事件。
- 待决策 resolve 已有归属校验和幂等抢占，防伪造 decision id、重复刷后果。
- 每日待决策预算、48 小时过期兜底、不可逆高牵挂拒绝自动兜底已接入后端。
- WebSocket 可推送 fate inbox / echo，前端不必只靠轮询。
- SeedVillage 已可在 onboarding 生成 20 人关系网，兑现“她身边有人”的承诺。
- 命运四槽已接入主指挥客户端侧栏，不再只存在于独立 `#fate` 入口。

判断：**命运开盒的最小闭环已经能跑，但还不是最终好玩的形态。**

### 5. 归因、抗命、惊喜门控

已完成：

- LLM 决策 payload 支持 attribution。
- 决策前构造人格、压力、记忆、关系快照。
- 归因不合法时可优雅回退安全决策。
- OOC 率遥测与自动降级护栏。
- 突然恋爱、卖传家宝、叛变等高戏剧性动作接入 `GateSurprise`。
- 抗命路径有溯源与成长旁白，避免玩家把“她不听话”理解成 bug。
- 玩家托梦 / 干预可成为后续回响前因。
- 主客户端已增加抗命溯源卡渲染，避免把机器编码直接裸露给玩家。

判断：**“意外但合理”的技术防线已经落地一大半。**

### 6. 大世界与离线自治地基

已完成：

- `worlds` / `world_members` / world tick / world bus。
- `cross_events` 作为跨玩家 append-only 唯一事实源。
- `units` 作用域列：world / region / life_state / last_active_tick。
- `agent_wake_queue` / `agent_decision_jobs` 持久化调度队列。
- RegionRunner 骨架：调度、worker、reclaim、panic 恢复、healthz 遥测。
- HOT / WARM / COLD 分层唤醒。
- 离线 ambient 动作：觅食、休息、社交、反思。
- HOT 单位可走 LLM 离线决策；失败、预算耗尽或超时会回退反射。
- MySQL 真实集成测试覆盖了关键方言路径。

判断：**后台大世界发动机的低层基本建好了，但角色还没有真正复杂的长期人生目标。**

### 7. PvE 威胁与 Boss

已完成：

- elite 遭遇：撞见、战斗、多回合消耗、胜利分赃、失败惩罚、命运卡。
- region-runner 威胁触发：已能按 HOT 单位、锚密度和确定性 roll 触发 elite。
- 锚加权威胁：危险更容易出现在角色在乎的地方，但保留破圈下限。
- 野外 Boss：组队多回合、按贡献分赃、失败各自惩罚。
- 世界 Boss：共享血池、异步出手、贡献账本、结算闩锁和并发硬化。
- 战斗/遭遇写状态时已做乐观并发硬化，降低覆盖其他写入的风险。
- 主指挥客户端已有手动触发 elite/PvE 遭遇入口，能把 PvE 结果刷新到界面。

判断：**PvE 原型链路很完整，但自然生成和玩法内容还不够系统化。**

### 8. 跨玩家与社会关系

已完成：

- 世界总线 → 命运收件箱桥接。
- 死亡事件世界化：一个角色之死会路由给在乎她的人。
- 跨玩家交互写入闭环。
- 社会客体撮合：common enemy、marriage pact、alliance 等可用 `MatchScore + arbitration` 选人绑定。
- 七种交互：结识、结盟、交易、联姻、反目、复仇、开战。
- `consent_gate` 三档异步同意；accept/reject 事务化，避免“状态已接受但效果没落”的坏账。
- ops token 已套到敏感写接口；未配置 `QUNXIANG_OPS_TOKEN` 时仍按原型开放。

判断：**跨玩家原语已经齐，但还不是面向玩家的完整社交玩法。**

### 9. 产品验证与运营观测

已完成：

- 假门 landing 页面。
- `POST /api/leads` 留资/问卷/事件落库。
- `GET /api/ops/leads-funnel` 漏斗聚合。
- `GET /api/ops/cost-dashboard` LLM 成本、token、fallback、活跃会话、单位经济聚合。
- AARRR 埋点骨架：角色创建、待决策入箱、待决策处理、玩家干预等。
- 漏斗事件常量补齐了账号注册、宪章完成、收件箱打开、状态卡查看、分享发起、购买、合规拦截等关键点。
- ops 类接口支持 `QUNXIANG_OPS_TOKEN`。
- 高危举报、审计、隐私擦除、保留期清理接口已套 ops token。

判断：**能开始做早期验证，但还没有完整增长/留存/变现看板。**

### 10. 商业化与合规最小纵切

已完成：

- `internal/billing`：SKU、购买、权益、计费流水、IAP 收据存证。
- 账户级 LLM 配额表 `account_llm_quota`，以月卡成本上限为默认 cap。
- `/api/billing/skus`、`/api/billing/purchase`、`/api/billing/quota/:accountId` 端点。
- 购买和配额端点从鉴权 token 取 account id，忽略客户端传入账号，避免越权。
- `internal/compliance`：实名状态、出生日期、未成年模式、宵禁、90 分钟防沉迷时长。
- `/api/compliance/verify`、`/api/compliance/gate/:accountId` 端点。
- 相关表已补 SQLite/MySQL 双驱动 schema。

判断：**商业化/合规从“只有文档”推进到“flag-gated 最小纵切”，但真实支付网关、默认启用策略和发行级合规流程还没完成。**

## 部分完成但还没真正闭环的部分

### 1. 命运开盒前台体验

已有四槽 UI 和真实后果，但还缺：

- 待决策不是情境化 Copilot 选项，仍是固定三键：由她去 / 疾呼拦住 / 默默看着。
- 后端已有倒计时/过期兜底，但主客户端还没有把倒计时和过期解释做成强可见体验。
- 高光卡没有“意料之中 / 意外但合理 / 太离谱”轻反馈。
- 即时人格快照仍偏 biography 展示，不是真正的“15 秒压缩快进微选择”。
- 缺少会话收尾卡和下次回访钩子。
- 缺少可分享死亡/高光传记卡。

结论：**能看命运，但还不够让玩家每天想回来开盒。**

### 2. 角色长期动机与离线宪章

已有 personality、memory、relationship、ambient 动作，但还缺：

- 六维 Ambition：power / vengeance / wealth / lineage / mastery / freedom。
- `Directive.Scope = offline_charter` 的正式离线宪章。
- 红线结构化入库，并作为 attribution 的 `redline` 前因。
- Freeze List：高代价离线动作自动上交待决策，而不是直接落地。
- pinned 物品字段与“传家宝绝不自动卖”完整链路。
- 周期性目标重估 `reassessGoal`。
- 人格漂移机制。

结论：**角色会活，但还没有足够强的“人生北极星”。**

### 3. 大世界威胁刷新

已有锚加权 elite 威胁，但还缺：

- `region.threat_level` 随世界事件累积。
- freshness 反扎堆项。
- 区域级 `arbitration.Resolve` 选址。
- field boss / dungeon 分段异步推进。
- world boss 从世界事件自然升级，而不是主要靠 HTTP 投放。
- 威胁 provenance 链：说明“为什么这场灾祸会发生在这里、牵动她”。

结论：**世界会刷危险，但还不像一个自洽演化的灾厄生态。**

### 4. 跨玩家玩法

已有世界总线、社会客体、七类交互和 consent，但还缺：

- 关系图传播与共享历史证据链。
- 威胁 party 自动撮合 social_object。
- backfill NPC / 离线玩家降速等组队兜底。
- 黑吃黑 `CROSS_BETRAYAL` 的受害者回应卡。
- 世界 Boss strike 的玩家权限、行动力/频率约束。
- consent 档位接入威胁加入、结盟背叛等更多主链路。

结论：**跨玩家事实源有了，跨玩家“玩法”还需要再长一层。**

### 5. 前端产品化

已有两个前端入口：

- 旧战棋客户端：功能多，但 `App.tsx` 非常大，维护压力高。
- Fate 客户端：方向对，但视觉、交互、反馈、内容密度仍是 MVP。
- 主客户端已经能打开命运侧栏、显示抗命溯源卡、手动触发 elite/PvE 遭遇。

还缺：

- 更清晰的新手首屏与第一分钟体验。
- 更强的高光卡视觉表现。
- 分享卡、传记卡、世界史回放。
- 前端 lint 清零。
- `PixiBoard` 大 chunk 拆分或调优。

结论：**能用，但离“让人愿意截图传播”的完成度还远。**

## 尚未完成的关键部分

### 1. 商业化系统

已有最小纵切，但未完成生产闭环：

- 真实 Apple / Google / 微信 / 支付宝等收据校验网关。
- 订阅续费、退款、撤销权益、渠道对账。
- 天命点的玩家可见资源表达。
- 付费档 provider / tick 频率 / 记忆容量配置。
- LTV / CAC / ARPPU / 付费漏斗聚合。
- 账户级配额与会话级 LLM 预算护栏的强制联动。

这是 PRD 里的商业闭环核心，目前已经有数据库和服务层纵切，但还不能当真实支付系统上线。

### 2. 发行合规

已有最小纵切，但未完成发行闭环：

- 内容安全 chokepoint 已有，但默认关，规则也仍偏基础。
- 年龄、实名、未成年人模式、宵禁、防沉迷表和端点已有，但默认 flag 关。
- 恋爱、生育、暴力等高压内容的产品级开关未做。
- ToS 里的“角色行为下限契约”未产品化。
- AI 安全红队与发行 gate 未建立。

如果要公开测试，至少需要先完成一版最小合规门。

### 3. 内容生产管线

尚未完成：

- 40+ 命运母题库：触发条件、两难结构、Copilot 选项、过期兜底。
- data-driven `translation_templates`。
- 赛季主题、神话层、burn-in 活史。
- 可运营的世界事件/GM 工具。
- 内容回归 golden replay。

现在系统能生成事件，但还没有稳定的内容工业化管线。

### 4. 大规模在线部署能力

已经有很多地基，但还缺：

- region 分片水平扩展（leader 选举 / region→实例稳定哈希 / rebalance）。**【仍真缺】**
- ~~`region_leases` 抢租原语已实现，但还没有接入 runner 主循环做多实例协调。~~ → ✅ **已接入**：`schedulePass` 按 `AcquireLease` 抢租、只处理持租区、每 pass 续租、退出释租（`regionrunner.go:295/341/737`，受 `QUNXIANG_REGION_LEASES` flag）。剩余是"对称抢租"无稳定归属，仍需 leader 选举/稳定哈希。

- Chronicle 物化视图与归档。
- 生产级监控、告警、压测、回滚策略。

### 5. 工程门禁清零

当前验证结果：

| 命令 | 状态 | 说明 |
|---|---|---|
| `cd backend && go build ./...` | 通过 | 后端可编译 |
| `cd backend && go test ./...` | 通过 | 后端单元/集成测试通过 |
| `cd backend && go vet ./...` | 通过 | `trade.go` 历史 unreachable 已清理 |

| `cd frontend && npm run lint` | 未通过 | 6 个 error、11 个 warning；阻断项仍在 `App.tsx` / `bootstrapPixi.ts` |

结论：**后端主门禁已经接近清零，只剩 statuslint 一处；前端 lint 仍未到可发版标准。**

## 当前最像“完成”的游戏切片

如果今天要拿给人看，最稳的演示切片是：

1. 进入 Fate 模式，创建一个角色。
2. 展示她的状态卡、命运高光、托梦。
3. 后端触发 fate event / elite encounter / worldized death。
4. 主指挥客户端打开命运侧栏，实时收到命运卡。
5. 玩家处理待决策，看到 morale / loyalty 真实变化。
6. 在主客户端触发一次 elite/PvE 历练，看到 HP/士气/钱包和命运卡变化。
7. 打开 ops dashboard 看 LLM 成本和活跃指标。

这个切片能证明项目的核心技术方向：

- 角色不是玩家直接控制的棋子。
- 世界事件会被翻译成她的命运。
- 玩家像祖魂一样影响她，而不是遥控她。
- 后果通过确定性代码落地，而不是 LLM 随口编。

## 下一轮优先级建议

### P0：先补“玩家为什么每天回来”

优先做：

- 高光卡轻反馈：意料之中 / 意外但合理 / 太离谱。
- 待决策 Copilot 化：每张卡生成 2 个情境选项，带倾向与后果提示。
- 把后端已有倒计时/过期兜底做成前端强提示和回访钩子。
- 第一段即时人格快照：创建后立刻给一个微选择高光，而不只是 biography。
- 会话收尾卡：今天你影响了她什么，下次她将面对什么。

理由：这比继续堆后台系统更直接决定留存。

### P1：补离线宪章与红线

优先做：

- `Directive.Scope = offline_charter`。
- 红线结构化存储。
- attribution 接入 `redline`。
- Freeze List。
- pinned item。

理由：这是“自治但不恶心玩家”的安全阀。

### P2：补商业和合规最小闭环

优先做：

- 接真实支付/收据网关，替换 stub。
- 把账户级 LLM quota 接成硬护栏，而不是只做旁路累加/查询。
- 内容安全和合规门的灰度配置。
- 年龄分级/未成年人模式对恋爱、生育、暴力内容的实际开关。
- 公开测试前的 ops token / auth 配置清单。

理由：不一定马上收费，但必须知道规模化后不会越跑越亏、也不会踩发行红线。

## 总体成熟度判断

| 维度 | 当前成熟度 | 判断 |
|---|---:|---|
| 后端模拟内核 | 75% | 战棋、状态、LLM、结算、事件留痕都较完整 |
| 命运开盒前台 | 55% | 四槽 MVP 接进主客户端，乐趣钩子还弱 |
| 大世界后台 | 55% | 调度和地基强，长期动机与复杂社会还弱 |
| PvE / Boss | 60% | 单链路扎实，自然生态未成型 |
| 跨玩家 | 45% | 原语完整，玩法层不足 |
| 商业化 | 35% | 有 flag-gated 纵切，缺真实支付和强制配额 |
| 合规发行 | 45% | 有实名/未成年/防沉迷纵切，缺默认启用和内容分级联动 |
| 前端产品化 | 45% | 主客户端接了命运/PvE/抗命，但未到传播级体验 |
| 工程门禁 | 82% | 后端 build/test/vet 绿，只剩 statuslint 1 处；前端 lint 仍红 |

最终判断：

**这个项目的“技术地基”仍然领先于“可玩产品外壳”，但这轮已经把命运面板、抗命卡、PvE 入口、商业化/合规纵切往前推了一截。下一阶段最该集中火力的是：把命运开盒的一分钟体验做成玩家每天愿意回来看的东西，同时把最后一处 statuslint 和前端 lint 清掉。**
