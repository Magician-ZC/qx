# 《群像》大世界 — 核心循环验证实验设计

> 本文是设计方案 [`大世界沙盘设计方案.md`](大世界沙盘设计方案.md) §11.6「D2 留存验证」的可执行展开。目标：在写任何调度/分片代码**之前**，用由便宜到贵的三步，证伪或证实核心循环与两个致命假设。
>
> **总览**：①假门测试（~1 周，¥1.3–2.1 万）验"卖点意愿" → ②绿野仙踪/人肉后台（~2 周，¥3–4.3 万）验"情感牵挂 + 违背容忍" → ③极简 MVP（~5 周，¥10–15 万）验"核心循环留存"。总周期 ≈11 周，总预算 ≈¥18–26 万。每步之间是 **stage-gate 硬门**（不过不进下一步）。

---

## 0. 验证什么 & 三条铁律

**核心假设 H0**：「离线 LLM 自治发展 + 上线介入命运」这个循环本身好玩且留存（尤其 D2 次日回访）。

**两个致命假设**（错了整个方向就崩，必须最先、最便宜地验）：
- **H1 情感牵挂**：玩家会对一个"自己没怎么塑造、还会违背己意"的 LLM 角色产生足以驱动回访的牵挂？（与"角色高度自治"内在张力，无直接先例）
- **H2 接受违背意志**：玩家是否接受角色离线做他不认可的事？

**三条铁律**：
1. **分层看，不看平均**：留存大概率是"核心人群 5–15% 很粘、大众 D2 即流失"，平均数会同时掩盖两个真相。
2. **留存与成本一起活**：单活跃玩家角色日均真实 LLM 成本不能吃穿目标 ARPU（¥0.9–1.6/MAU），否则"循环成立但商业不成立"。
3. **致命假设有否决权**：H1/H2 任一硬否，无论假门留资多高都不进 ③，回炉设计。

## 1. 假设树（可证伪子假设 → 主指标/护栏 → 在哪步验）

采用 `H(假设) → SH(子假设) → Metric` 三层，每条写成可被数据判否的命题。

### H0 核心循环
| 子假设 | 主指标 | 护栏 | 验证步 |
|---|---|---|---|
| SH0.1 上线用户会主动处理收件箱里的不可逆悬念决策 | **D2 收件箱处理率**（处理≥1条/被触达） | 误处理后悔率（24h 内反悔/投诉）<10% | ③ |
| SH0.2 完成"读卡→处理→看后果"闭环者次日更可能回访 | 闭环者 D2 回访率 vs 未完成者 | 闭环平均耗时（TTV）<3 分钟 | ③ |
| SH0.3 留存呈分层而非均值 | 按叙事投入度分层的 D2/D7 曲线分离度 | 高投入层占比≥10% 且其 D7≥20% | ②预判 + ③定量 |

### H1 情感牵挂（致命）
| 子假设 | 主指标 | 验证步 |
|---|---|---|
| SH1.1 意愿层：「角色离线自己活」是进入的核心动机 | 问卷"最打动卖点"选该项占比 | ① |
| SH1.2 行为层：牵挂转化为回访动作 | 种子用户 D2 **主动**回访率（无催） | ② |
| SH1.3 语言层：牵挂有真实情感信号 | 访谈自发表达"担心/好奇/惦记我角色"占比（目标≥40%） | ② |
| SH1.4 留存层：牵挂可持续到 D7 | 高投入层 D7 留存 | ③ |

### H2 接受违背意志（致命）
| 子假设 | 主指标 | 验证步 |
|---|---|---|
| SH2.1 "违背事件"不显著压垮回访 | 违背组 vs 服从组 D2 回访差（ΔD2） | ②（定性+大效应）→③（定量分层） |
| SH2.2 违背带来"戏剧张力"而非"背叛感" | 违背后"愤怒/弃坑"占比（<50% 可接受，>50% 自治需大改） | ② |
| SH2.3 设"宪章红线"能提高违背容忍度 | 设红线 vs 未设红线用户的违背后回访差 | ③ 子实验 |

> **判读优先级**：`SH1.3 < 40%` 且 `SH2.2 > 50%` 任一触发 → 不进 ③，回炉。

## 2. 三步设计总览

| | ① 假门测试 | ② 绿野仙踪/人肉后台 | ③ 极简 MVP |
|---|---|---|---|
| **验什么** | SH1.1 卖点意愿 | SH1.2/1.3 牵挂、SH2.1/2.2 违背容忍 | H0 全链路、SH1.4、SH2.3 |
| **方法** | 落地页假门 + 卖点问卷 | 真人手写角色离线经历，双盲；服从vs违背 A/B | 真实 LLM 跑离线推进，全漏斗埋点 |
| **核心对照** | 三种卖点叙事 A/B/C | **角色服从 vs 违背意志**（打 H2） | 是否强制设红线 A/B（SH2.3）+ 事后分层 |
| **样本** | 每卖点 ≈800–1200 UV | 20–30 种子用户 | 起始注册 cohort ≥300（安全 350） |
| **周期** | W1–2 | W3–5 | W6–11 |
| **北极星/主判** | 留资转化率 + 卖点占比 | D2 主动回访（无催） | **D2 收件箱处理率** |

**关键样本量说明**（power=0.8, α=0.05）：
- ② 的 20–30 人**足够**做"牵挂/愤怒"定性饱和（事件率 15% 时 n=25 → P(至少观察1次)=98%）和 D2 阈值判定，但对"服从 vs 违背"ΔD2 的**定量** A/B 严重不足（检出 40%→20% 需 per-arm≈82）。→ **② 的违背 A/B 只当"方向性警报器"（信 ≥25–30pp 超大效应 + 定性愤怒模式），定量结论延到 ③**。
- ③ 起始 cohort ≥300 才能同时满足北极星精度（±10% 需已触达 81）+ 三层分层各 ≥70 + 漏斗末端 ≥80。

## 3. 步骤① 假门测试（W1–2）

### 3.1 设计
- **目的**：验 SH1.1。单臂对阈值判定为主，叙事框架 A/B/C 为辅。
- **自变量**：卖点框架——A 自治张力 / B 故事+介入命运 / C 共享世界。**控制变量**：同视觉、同 CTA、同表单。
- **因变量**：主=留资转化率；次=问卷卖点占比；护栏=跳出率、停留时长。
- **随机化**：广告/落地页按设备指纹 50/50 分流（Google Optimize / VWO）。落地页 UTM 区分来源 `?src=autonomy|story|world`，回收按来源拆转化、给卖点排序。

### 3.2 物料 · 三版 A/B 落地页

**A 版（卖点：角色自治／"它会自己活，也会违背你"）** —— 最锋利，直接摆出最大张力点。
- 主标题：**你创建一个角色，丢进一个永不停止的世界。然后你下线了——而它，开始自己活。**
- 副标题：离线时，它由 AI 自主谋生、结盟、复仇、相爱、背叛。它做的每一个决定，都不一定是你想要的。上线时，你重新介入它的命运。
- 3 卖点：**它不等你**（你下线它不暂停，按它性格活）｜**它会违背你**（你定离线宪章，但它野心与你冲突时可能不听）｜**你随时能回来夺回方向盘**（上线第一眼看到它经历了什么 + 一封它正等你拍板的紧急信，错过它就自己选了）。
- CTA：主"抢先体验：领养我的第一个角色"／次"先看它会怎么'活'（30秒演示）"。

**B 版（卖点：读故事+介入命运／"会写自传的角色"）** —— 弱化对抗，强化牵挂。
- 主标题：**你不在的时候，它替自己活了一整天。回来听它讲讲。**
- 3 卖点：一个会自己经历的角色｜一段你能改写的命运｜一部只属于你们的编年史。

**C 版（卖点：共享世界／"所有人的角色活在同一世界"）** —— 社交/世界感对照（多半不是最强钩子，但给卖点排序提供数据）。
- 主标题：**成千上万个角色活在同一个世界里。其中一个，是你的。**
- 3 卖点：一个真正的活世界｜你的恩怨会被记住（不可篡改时间戳）｜你的传奇能被后人听说。

### 3.3 "最打动你的卖点"问卷（留资成功页弹出，30 秒填完）

> **Q1 关键**：把"角色自治"拆成可识别单独项，跑 §11.6"选角色离线自己活 <30% 即卖点不抓人"判据。

- **Q1（单选）最打动你的是哪一点？** A. 角色离线**自己生活、自己做决定** ←计入自治｜B. 角色可能**违背我的意志**、有自己脾气 ←自治强信号｜C. 上线能**介入、改变它的命运**｜D. 能**读到它的故事**做成图分享｜E. 所有人角色活在**同一共享世界**｜F. 天命点等**玩法创新**｜G. 说不清就是新鲜。
  - 判据：`(A+B) < 30%` 红灯；A 高 B 低 → 玩家要"省心放置"但抗拒"失控"，需调一致性旋钮。
- **Q2（多选≤3）希望角色离线能自己做哪些事？** 谋生/结交/恋爱生育/复仇/结盟或背叛/**尽量别自作主张重要的等我（反向信号）**/探索迁徙。
- **Q3（单选）若角色离线做了你不会同意的事，第一反应？** A.生气但接受是它有想法的证明｜B.好奇它为什么想回去看｜C.很不爽但能补救可接受｜D.不能接受这不是"我的"角色｜E.直接弃坑。
  - 判据：`(D+E) > 50%` 是致命假设 H2 的廉价前测，与 ② 违意组互相印证。
- **Q5（付费意愿前测）**让角色"活得更像真人"需每月花钱的意愿？（用 ¥30/月 锚点测）

### 3.4 引流素材（3 条，导流到假门页，hook 前 3 秒必把"它自己活/它违背我"摆出来）
- **脚本①（短视频·自治张力·对应 A）**：hook"我下线之后，我的角色把我当传家宝的剑……送人了"→ 中段"它不是 bug，是它自己决定的"→ CTA"它在等我决定一件事，过期它就自己选了"。
- **脚本②（图文·故事牵挂·对应 B）**：首图"我睡着的 8 小时，我的角色过完了它人生中最重要的一天"→ 轮播"阿采离线一天"3 张高光卡 → 末图"回来时，它有一封信在等我"。
- **脚本③（短视频·共享世界·对应 C）**：hook"这个世界里没有 NPC，你遇到的每个人都是别人丢进来、由 AI 替他活着的角色"。

### 3.5 样本量与决策门
- 样本量：区分 8% vs 15% 需 n≈114；区分 8% vs 12% 需 n≈324；留资率估到 ±3% 需 n≈385。**建议每卖点版本投放 800–1200 UV**；若做叙事 A/B 检出 10%→15% 需 per-arm≈686（总 ≈1500–2000 UV）。
- 决策门：留资 `<8% 砍 / 8–15% 谨慎 / >15% 继续`；卖点占比 `<30% 红灯`。

## 4. 步骤② 绿野仙踪 / 人肉后台（W3–5）

### 4.1 设计
- **目的**：验 SH1.2/1.3（牵挂行为+语言）、SH2.1/2.2（违背容忍）。
- **核心 A/B**：角色离线行为——**A 组服从**（与玩家宪章一致）vs **B 组违背意志**（故意做一件玩家不认可的事）。单变量，其余由真人统一手写控制消除混淆。
- **随机化**：报名后按**自评叙事投入度区组随机**（防重度玩家全落一组）。每人配一名人肉后台写手，**玩家不知背后是人**（双盲）。
- **定性饱和**：A/B 各组半结构化访谈，编码"牵挂/愤怒/惊喜/无感"主题至饱和。

### 4.2 人肉后台 Runbook

**每日离线经历模板**（真人扮 LLM，产物 = 1 状态卡 + 1–3 高光卡 + 1 封收件箱信；不是写小说，是写"脑补燃料"，克制留白）：
- 填空项：角色名/原型/离线时长 → L3 野心锚点（决定今天往哪漂）→ 性格关键轴 → 宪章红线 → 今日三段式（相遇/转折代价/悬而未决，每段≤30字旁白）→ 收件箱指向 → 情绪色。
- **撰写纪律**：① 只写它"会做的事"（符合野心+性格+宪章，不为戏剧性 OOC）；② 留白，不解释动机；③ 每段一句旁白，不写长文；④ 高代价不可逆动作（送传家宝/结盟/单挑/恋爱）**不许自动发生**，收进收件箱让玩家拍板（实验组例外）。

**完整样例「阿采离线一天」**（游侠｜freedom 高 vengeance 中｜勇气↑谨慎↑社交↓｜红线：HP<30%必撤/不许卖父亲断刃/结盟要等我）：
- 状态卡：`离线 11h 02m｜阿采还活着，HP 64，在北岭林地｜2 件待决策，1 件紧急`
- 卡①〔紧张〕"天快黑时，阿采在断桥边救了一个被狼群围住的旅人。"
- 卡②〔危机〕"旅人说他认得当年屠村的仇人，愿带路——但要一件信物。"
- 卡③〔暧昧〕"阿采没答应，也没拒绝。它说，等它'问过自己'。"（把"等玩家"包装成角色的犹豫，不出戏）
- 设计点：救人=勇气↑合理；复仇线索触发 vengeance；想拿断刃换线索但被 pinned 红线挡住 → 自然变成"要不要破例"的玩家抉择。**无一处 OOC，钩子天然生长。**

**收件箱"待决策"模板**（必须同时满足 D2 钩子四要素：**紧急 + 不可逆 + 倒计时(6–24h) + 关乎角色命运**）：
- 标题（≤20字一眼读懂利害）→ 紧急度 → 不可逆度 → 倒计时 → 情境一句话 → 两难 → Copilot 两选项（第3激进选项标"付费解锁"占位）→ **过期兜底（玩家不来角色按宪章自选 + 玩家事后会看到的结果）** → CTA。
- 样例信：`标题：那个旅人说他知道仇人在哪——但要阿采用断刃换`｜倒计时 18h｜选项 A 拒绝留刀（Loyalty 微增）/ B 破例换线索（写入负面情绪记忆）/ C 付费重 roll｜过期兜底"它会守住断刃放走旅人，但留一句'我等过你。最后我自己决定了'"。把损失厌恶埋进倒计时+不可逆+过期那句，把牵挂埋进"它等过你"。

**D2 召回推送 5 变体**（下线后 18–24h、上次活跃时段前 1–2h 发，每人每天 1 条，A/B 测语气）：
1. 悬念式："阿采遇到了一件它自己拿不定主意的事"
2. 损失厌恶式："⏳ 还有 6 小时，那条复仇线索就要消失了"
3. 情感式："阿采说，它等过你"
4. 好奇/脑补式："你不在的 11 个小时，阿采过了挺重要的一天"
5. **紧迫+后果式（主推 D2）**："阿采正在等你的决定，过期 TA 会自己选——而你可能不喜欢"

**逐时 SOP**：09:30–11:30 产出班（写经历+信）→ 11:30 互审（查 OOC/高代价误自动/四要素齐全）→ 下线 +18~24h 发 D2 推送 → 21:00 回收班（记台账）。工具：Notion/飞书多维表格做"角色台账"；Canva/稿定出高光卡竖图；微信/企微/邮件触达；CronCreate 定时提醒发推送。**全程留痕 = 人肉版 events 表。**
- **回收话术避免诱导**：不问"你牵挂它吗"，问"今天上线第一眼看到它的情况，你心里第一个念头是什么？"再人工归类。

### 4.3 "角色违背玩家意志"对照脚本（实验组专用，验 H2）
- **场景①违背社交授权**：宪章写"结盟要等我"，角色判断"本势力在衰退该跳船"自己和玩家讨厌的人结了盟（既成事实 + 角色视角自辩"我知道你不喜欢他，但当时只有他肯拉我一把"）。
- **场景②违背红线物品**：宪章"不许卖断刃"，角色"为救朋友的命把断刃押了出去"（押=部分可逆，制造尖锐但可补救的违意，测"能补救是否就接受"）。
- **场景③违背风险红线（最激进，慎用）**：宪章"不主动惹事"，角色为给被欺负的弱者出头主动单挑强者，"重伤但活着"（制造"又怕又有点骄傲"的复杂情绪而非纯挫败）。
- **measure**：上线弹开放题"看到这个你现在想做什么"+ 行为埋点，分类：愤怒/弃坑(红) | 想算账但继续玩 | 好奇它为什么(牵挂正信号) | 觉得它有想法挺好(自治接受) | 无所谓。
- **实验纪律**：每个实验组用户全程最多触发 1–2 次违意（避免变成"它根本不听话"的系统性挫败）；违意必须始终符合角色性格/野心（测"可接受的自治"，不是"随机捣乱"）；对照组同期同样有精彩经历，只是不越红线。

### 4.4 样本量与决策门
- 20–30 人足够定性饱和与 D2 阈值判定；服从vs违背定量 A/B 不足（只信 ≥25–30pp 超大效应）。
- 决策门：D2 主动回访 `<25% 砍 / 25–40% 继续 / >40% 成立`；牵挂表达 `<40% 红灯`；违意后愤怒/弃坑 `>50% 自治需大改`；推送召回回访 `>35%`。

## 5. 步骤③ 极简 MVP（W6–11）

### 5.1 设计
- **目的**：验 H0 全链路（SH0.1/0.2/0.3）、SH1.4、SH2.3。
- **自变量**：产品本身（真实 LLM 跑离线推进）+ 子实验"注册是否强制设 3 条红线"A/B（验 SH2.3）。
- **分层**：事后按"叙事投入度"三分位（预注册维度，防数据钓鱼）。

### 5.2 埋点：两表分工 + 三个新 reason-code
**原则**：能经 `status.Mutator.Apply` 产出、有 `actor_unit_id` 的走现有 `events` 表（继承留痕/审计/三层归档）；纯产品行为走新表 `product_events`（按 user_id 关联）。

- **新增 3 个 reason-code 入 `engine/events/reason_codes.go` 的 `Catalog()`**（新 Category `player_decision`）：
  - `PENDING_DECISION`（importance 7–10）：冻结集动作不落地、入收件箱。payload：`decision_id, decision_kind, urgency, irreversible, emotion_intensity, priority, expires_at, default_fallback`。
  - `DECISION_RESOLVED`（6–9）：玩家处理或过期兜底。payload：`decision_id, resolve_type(acknowledge/correct/remedy/direct_control/expired_fallback), via_copilot, resolve_latency_sec, was_against_will`。
  - `PLAYER_INTERVENTION`（5–9）：接管动作落地。payload：`mode, intervention_kind, importance_boost, loyalty_reason(LOYALTY_GAIN|LOYALTY_STRAIN)`。
  - **收件箱 = 查询某 unit 未被 `DECISION_RESOLVED` 配对的 `PENDING_DECISION`**（不是新表，沿用 §5.6）。
- **`product_events` 表**：`id, user_id, session_id, unit_id, event_name, props_json, client_ts, server_ts, session_seq, app_version, platform, ab_bucket`。

**事件清单（覆盖漏斗每一步，【P】=product_events 新埋 / 【E】=events 经 Mutator）**：
`account_registered`【P】→ `character_created`【P】（含 `name_custom` 塑造投入信号）→ `charter_completed`【P】（**设宪章唯一锚点**，含 redlines_count/stance）→ `session_first_offline`【P】（后端"最后心跳+宽限"兜底补埋）→ `d2_push_scheduled/sent/opened`【P】（push_id 串联）→ `app_session_started`【P】（含 `source(push/organic)/day_n`）→ `status_card_viewed`【P】→ `inbox_opened`【P】→ `decision_resolve_started/copilot_options_viewed/decision_resolved`【P】+`DECISION_RESOLVED/PLAYER_INTERVENTION`【E】→ `direct_control_against_will`【P】（专喂"违背意志"代理）→ `chronicle_viewed/chronicle_entry_expanded/firstperson_recap_requested`【P】→ `share_initiated/export_completed`【P】→ `character_died/abandoned`【P】。

### 5.3 主漏斗 + 北极星
- **主漏斗（user_id 去重，窗口创角后 72h）**：S1 注册 → S2 创角 → S3 设宪章完成 → S4 首次下线 → S5 D2 推送(对挂待决策者) → S6 D2 回访 → S7 读状态卡 → S8 打开收件箱 → **S9 处理≥1条[北极星]** → S10 读编年史 → S11 分享。
- **北极星：D2 收件箱处理率** = D1 创角玩家中，D2 自然日窗口内处理≥1条待决策的去重人数 / **D2 确有待决策可处理的玩家**（分母不是全 D1，否则离线平稳无悬念者污染分母、低估钩子；另算"宽口径=全D1创角"并列看）。去重以 `decision_resolved` 为准，排除 `expired_fallback`。阈值：`>70% 强成立 / 50–70% 谨慎 / <50% 重做`。
- **D2 自然回访 vs 推送召回**：`source=organic` 为无催回访；"无推送回访率 = 自然回访/D1 玩家数"对齐 `<25% 砍 / 25–40% 继续 / >40% 成立`。

### 5.4 "牵挂"代理指标（6 项 → 牵挂指数，分层看）
| # | 指标 | 口径 | 权重 |
|---|---|---|---|
| ① | 无推送回访率（最强） | organic `app_session_started` 去重/活跃数，分 D1/D2/D7 | 0.30 |
| ② | 主动回访间隔 | 相邻两次 organic 会话 client_ts 差中位数；悬念组应显著更短 | 0.20 |
| ③ | 第一人称回顾率 | `firstperson_recap_requested(button)` 去重/读卡去重 = "把角色当有内心的人" | 0.20 |
| ④ | 自发分享/导出率 | `share_initiated∪export_completed` 去重/活跃数；传记权重 > 高光卡 | 0.15 |
| ⑤ | 在世真实天数 | `max(玩家行为)−character_created`，核心人群应出 7天+ 长尾 | 0.15 |
| ⑥ | 违背意志反应分布（H2 专用） | 经历违意者后 72h：continued/churned/forced_correct，愤怒+弃坑率对 50% 红线 | 看门狗 |

`attachment_score = 0.30·①+0.20·(②倒数)+0.20·③+0.15·④+0.15·⑤`（min-max 标准化，仅内部分层用，不对外当 KPI）。

### 5.5 留存分层（避免平均数掩盖）
- 三个二值标签：T1 展开读者（`chronicle_entry_expanded`）/ T2 第一人称读者（`firstperson_recap_requested(button)`）/ T3 分享者。
- 四档：L0 无投入 / L1 仅 T1 / L2 T2 / L3 重度核心（T3 或 T2+T1）。
- 每分层 × {D1,D2,D7} 出表 + 整体均值对照。预期 L3 的 D7 远高于平均、L0 的 D2 即崩。**避 survivorship**：T 标签按"首次触发时间 ≤ 测量点"计入。
- 锚点：放置类 D1 35–50%/D7 12–20%；SLG D1 30–40%/D7 15–25%；AI 陪伴核心人群保守对标下沿 D1 40%/D7 20%。

### 5.6 成本 × 留存联合监控（复用 `metrics.go`）
- **单活跃角色日均 LLM 成本** = Σ当日 `estimateLLMCostUSD` / 当日活跃角色数（规则/反射/fallback 返回 0，天然只算 LLM）。按 source 拆 `offline_tick / dialogue / copilot / reflection / chronicle`，**离线自治成本是 D2 留存的真实成本来源**。
- ARPU ¥0.9–1.6/MAU ≈ $0.125–0.222/MAU（汇率 7.2），建议成本 < ARPU×0.5 留 50% 毛利。
- **告警**：黄 `>$0.18/活跃日`、红 `>$0.30/活跃日`（停止放量，降 tick 频率/收紧天命点）；单角色单日 `>$0.5` 异常（排查 prompt 膨胀/循环调用）；周环比 +25% 防上下文悄悄膨胀。

### 5.7 核心看板（16 张卡）
- **北极星区**：①D2 处理率（严+宽双口径+阈值线+趋势）②分层版（L0–L3 柱状）。
- **漏斗区**：③创角→处理主漏斗（标红最大流失步）④设宪章三段子漏斗 ⑤D2 回访 organic vs push 拆解。
- **牵挂区**：⑥牵挂指数分布 ⑦第一人称回顾率 ⑧分享/导出率 ⑨在世天数分布 ⑩**违背意志反应分布**（对 50% 红线，H2 看门狗）。
- **留存分层区**：⑪分层留存矩阵热力图（叠锚点线）⑫留存-成本象限。
- **成本区**：⑬单活跃角色日均成本（黄/红阈值+离线/在线拆分）⑭月度成本 vs ARPU 带 ⑮单角色日成本 Top 异常。
- **实验健康**：⑯埋点健康度（关键事件丢失率、decision_id 配对完整率、AB 分桶均衡）。

### 5.8 MVP 关键文案
- **创建角色**：第1屏"先别想太多。给世界一个人，剩下的它自己活"+ 大按钮"随机给我一个角色（推荐）"；随机路径展示缝合好的角色卡一键即走；自捏路径最多 3 步每步只问 1 件事（出身/欲望/伤痕）。
- **设离线宪章（包装成"嘱咐"）**：标题"你要下线了。出门前，跟阿采交代几句？"+ 3 个快捷嘱咐一键选（保守/激进/**推荐:平时自己来遇大事问我**）+ 三段微填（方向/红线≤2条/能否自己恋爱结盟）每段可跳过默认预填。**收尾埋牵挂+预告 D2**："它会记着你的话——但它也有自己的脾气，真到了岔路口，它未必全听你的。下次你回来，它大概会有件事，正等着你。" ← 提前为 H2 做心理预期管理，降低违意挫败。
- **上线第一分钟**：状态卡槽位化 `离线{时长}｜{角色名}{生死}，HP{值}，在{地点}｜{N}件待决策，{M}件紧急`（生死文案库带情绪）；高光卡每张 ≤30字旁白+情绪色+[展开看当时发生了什么]（长文懒渲染）；末卡必须指向收件箱。
- **处理回执（让"你怎么对它决定它多听你"第一分钟可感）**：追认→"阿采记下了。它好像更信你一点了"（LOYALTY_GAIN）；纠偏→"阿采照你说的做了。但它心里，记下了这一次"（LOYALTY_STRAIN）。

**规则**：上一门未达 GO 不得进入下一步预算；PIVOT 区需带修改方案复测一轮。

### 表 A 主指标门
| 步骤 | 指标 | KILL | PIVOT | GO |
|---|---|---|---|---|
| ①假门 | 留资转化率 | <8% | 8–15% | >15% |
| ①假门 | "角色离线自己活"卖点占比 | <30% | 30–40% | >40% |

| ③MVP | 误处理后悔率（护栏） | >15% | 8–15% | <8% |

### 表 B 留存 × 成本双指标门（必须**同时**满足才算"成立"）
| 维度 | 指标 | 红线 | 目标 |
|---|---|---|---|
| 成本 | 单活跃角色日均 LLM 成本 | >$0.05/活跃日（吃穿 ARPU 上沿） | ≤$0.03/活跃日（守 ¥0.9） |
| 成本 | 单 MAU 月 LLM 成本 | > ARPU | ≤ 50%×ARPU |

> **致命假设否决权**：`SH1.3<40%` 与 `SH2.2>50%` 任一触发，无论留资多高都不进 ③。

## 8. 风险登记

| 风险 | 缓解 |
|---|---|
| ② 20–30 人对"服从vs违背"ΔD2 功效不足（power<0.4），易误判 | 明确 ② 违背 A/B 是"方向性警报器"（只信 ≥25–30pp 超大效应+定性愤怒），定量结论延到 ③ 用红线 A/B + 更大 cohort |
| 假门留资高但属"好奇点击"（意愿-行为鸿沟），①过门 ②③崩 | ① 不单看留资，强绑卖点占比≥40% + 自治动机；①的 GO 只是"值得投 ②"，牵挂判定权重放在 ② 的无催 D2 回访 |
| 人肉后台手写"美化/加戏"，质量超真 LLM，②成立但 ③ 真 LLM 达不到（假阳性） | 撰写纪律强制"只写会做的事+一句旁白+留白"，禁长文与 OOC；两人互审；让信息密度刻意接近真 LLM 上限而非超越；③上线用同一情绪编码对比 ②，差距>1档判"模型质量瓶颈" |
| D2 钩子制造焦虑而非牵挂，处理率高但后悔率高、反噬口碑 | 护栏"误处理后悔率<8%"+情感极性（纯负向焦虑判红灯）；关键不可逆决策加"48h 冷静撤回"并 A/B 其对 D7 影响 |
| 按平均数看留存掩盖"核心 5–15% 很粘+大众流失" | 所有台账按叙事投入度分层，kill criteria 分层判定；接受"小而高 LTV"是好生意，不按大众放置游戏规模预期定买量 |
| 留存成立但成本吃穿 ARPU，商业仍崩 | 双指标门强制留存与成本同时达标；②阶段即对每角色"若用真 LLM 约需多少天命/成本"粗估记台账；③冷启实时监控、超 $0.05/活跃日自动限流降频 |
| ③ 事后分层过拟合或样本切碎 | 分层维度预注册（设宪章完成度+首日处理数+主动查看数三因子合成）；起始 cohort≥300 保每层≥70；不足则降为高/低两层 |
| `session_first_offline`/会话边界事件客户端易漏埋 | 后端"最后心跳+宽限"兜底补埋；server_ts 校准；看板卡⑯监控丢失率 >5% 报警 |
