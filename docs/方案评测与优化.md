# 《群像》大世界 — 方案评测与优化

> 对 [`大世界沙盘设计方案.md`](大世界沙盘设计方案.md) 与 [`验证实验设计.md`](验证实验设计.md) 的六角度独立评审（制作人/发行、引擎架构、商业化/单位经济、市场/竞品、核心玩法、验证方法论）综合而成。结论汇总成一份**优化 backlog**，并据此产出两份可直接拿出手的方案：[`游戏开发方案GDD.md`](游戏开发方案GDD.md) 与 [`产品方案PRD.md`](产品方案PRD.md)。

---

## 0. 一页结论

**成熟度画像**：两份文档在「**验证纪律 + 引擎复用 + 设计自我红队**」上属种子期顶尖（§11 自承 P2W、§1.5 决策/结算分层、三步 stage-gate、双指标门、分层留存），可直接立项**先导验证**（¥18–26 万 / 11 周）。

**但它们还不是一份「可拿给投资人/团队/发行」的完整方案**——六个角度一致指出，它把 90% 精力投在「能不能留住人 / 烧不烧得起钱 / 工程能否复用」，却几乎没回答制作层与商业层的问题：

| 维度 | 现状 | 缺口（六角度共识） |
|---|---|---|
| 制作可行性 | 工程路线图有 | ❌ 无 Vertical Slice、无团队/人月/预算、验证 GO 后到上线是悬崖 |
| 商业模型 | 成本侧罕见地扎实 | ❌ 收入侧几乎空白：无天命配额闭合表、无 P&L、付费率 4–6% 大概率 2× 高估 |
| 市场/发行 | 几乎空白 | ❌ 无竞品/TAM/GTM/获客经济学/监管路径（版号+AI 备案是生死门） |
| 核心玩法 | 系统丰富 | ❌ 没定义 core toy；moment-to-moment 是「通知整理」不是「玩游戏」 |
| 内容/运营 | 隐含丢给 LLM | ❌ 无内容生产管线、无 live-ops、无 AI 内容安全 |
| 验证方法 | 投资级 | ⚠️ 北极星选错（处理率≠牵挂）、致命假设拖到最贵的 ③ 才触达真 LLM |

**一句话**：这是一份优秀的「**产品+实验设计**」，但要变成「**可发行的游戏 + 可融资的生意**」，还差 7 块——本评测把它们补齐为优化 backlog。

## 1. 各角度核心结论（诚实总评）

1. **制作人/发行**：可投「验证先导」，暂不可「立项做完整产品」。三大悬崖：①验证→上线的 scope/人月/里程碑/VS 完全缺失；②AI 内容安全/审核/合规/版号几乎没碰（品类级发行硬门槛）；③单位经济学缺位。
2. **引擎/技术架构**：架构判断对（「复用而非重写」主脊正确，四笔工程债定位经实读代码核实属实），但**系统性低估了三处「代码层重构」**——尤其「Mutator 把 SessionID 换成 RegionID = 零改动」与代码现实不符（Mutator 无事务、Save 全 blob 重写还内部再读）；「prompt 缓存」依赖仓库里不存在、需改 wire 协议的功能却被列为「立即可做」。缺一份**接口级技术规格**。
3. **商业化/单位经济**：成本侧扎实，**收入侧只有半张表**。天命配额（连接成本与收入的核心变量）从未定义 → 毛利/回本算不出。付费率 4–6% 与文档自己「大众 D2 即流失」的结论冲突，真实是「窄漏斗+鲸鱼」。P2W 仲裁 tick 在公平上对，但**砍掉了传统手游 ARPU 占比 60–70% 的战力/加速档，却没做收入重定向测算**。
4. **市场/竞品/定位**：几乎一张白纸，且最致命。**获客经济学倒挂**风险——以 ARPU ¥0.9–1.6/MAU 对上 midcore 全球 CPI $3.25–4.5，付费买量几乎必然 LTV<CAC，只能靠自然量，但这个生死假设全文未声明未验证。最像的活体对标 **Status by Wishroll**（2025/2「社媒版模拟人生」，两周破百万、50 万 DAU，靠 Inworld 砍 95% 推理成本才跑通）缺席。
5. **核心玩法/留存**：**没定义 core toy**。moment-to-moment 拆开看玩家只在做「读 3 张卡 → 点追认/纠偏 → 关 App」——这是**通知整理循环**，不是游戏循环，乐趣 99% 寄生在「牵挂」这一最未验证的假设上。更糟：§5.5 把唯一有手感的「直控」设计成要被惩罚限流的反模式，**亲手阉割了「玩的那一下」**。
6. **验证方法论**：方法论自觉超 90% 同类，但三步「递增保真度却不递增真实代价」——意愿-行为鸿沟、人肉后台对真 LLM 的保真度差，被推迟到最贵的 ③ 才暴露。北极星「D2 收件箱处理率」**高度可被倒计时/推送操纵、与牵挂近乎正交**，会给系统性假阳性。

## 3. 本轮已落地的引擎升级（O9 的一部分）

作为「对引擎中心升级」的首批落地，已在 `backend/internal/engine/` 写出**可编译、可测试、零破坏**的新核心，并为仓库补上**第一套测试**（此前 0 测试）：

| 交付 | 包/文件 | 内容 | 测试 |
|---|---|---|---|
| **三层执行模型·决策层** | `engine/decision` | `Router.Route(Situation) Decision`：安全反射(L1 护栏)优先 → 关键节点升级 LLM → 日常零 LLM。把「<2% 上 LLM」从「降级模式」扶正为「设计常态」 | 8 用例：安全反射抢占、关键节点升级、日常零升级、确定性 |
| **零和仲裁原语** | `engine/arbitration` | `Resolve(Contest) Outcome`：胜负仅由 Score + 确定性掷骰决定，**与频率/入队顺序无关**（§11.3 反 P2W）。A-Res 加权抽样保证「胜率∝Score」 | 6 用例：确定性、顺序无关、**频率不变**、**胜率比例**（70/30→0.70±0.04）、压倒性、边界。**测试当场捕获并修复了一个真实公平性 bug**（FNV 雪崩弱致高分必胜） |
| **批量状态写入** | `status.Mutator.ApplyBatch` | 把「每决策 ~15 次 DB 往返」收敛为「按单位读一次/写一次 + 单事务批量插事件」，与逐次 `Apply` 语义等价 | 5 用例：等价性、按序对齐、单事务落盘、未知码整批拒绝零副作用、空入参 |

`go build ./... && go vet ./internal/engine/... && go test ./... && go run ./cmd/statuslint ./internal/engine/...` 全绿。详见 [`游戏开发方案GDD.md`](游戏开发方案GDD.md) §引擎架构与 CLAUDE.md。

> 这三件直接对应优化项 O9：决策层接口落地、零和仲裁原语、Mutator 批写。它们是新增能力 + 对最热写路径的真实优化，且不触碰现有可玩性（纯新增 + 一个新方法）。剩余 O9（events 双键过渡、ReflexPolicy/Deliberator/Resolver 全接口收口、可玩性回归基线）列入引擎升级 spec 后续阶段。

## 4. 给决策者的三句话

1. **马上能做**：把上面 W0 预实验（<¥3 千）和北极星修正先做了——它们用最小代价堵住整套验证最大的假阳性漏洞。
2. **方案补全**：GDD 补 core toy + 首日啊哈 + 自治/掌控玩法解法 + VS 规格；PRD 补单位经济 P&L + 市场/GTM/获客经济学 + 监管合规 + 里程碑/人月。这两块补上，才是「可拿出手」。
3. **定位收敛**：在「AI 陪伴红海」与「叙事养成沙盘」之间**选定主货架 = 叙事养成沙盘**（确定性结算+不可篡改留痕是这里才能赢的壁垒），把陪伴当获客钩子而非身份——这决定卖点、定价、买量基准、监管路径四件事。
