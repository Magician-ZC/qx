# 《群像》大世界沙盘设计方案

> ⚠️ **定位修正（务必先读 [`设计宪法-角色命运开盒.md`](设计宪法-角色命运开盒.md)）**：本方案是「**后台发动机**」的设计，**不是前台玩法**。项目最高定调已改为「**角色命运开盒游戏**」——完整世界在背后运行，但**大世界不能直接扑到玩家脸上**，前台永远从一个角色的一段命运开始。本文描述的「无限 agent 大世界」是那台发动机；玩家真正玩的、以及前台如何只呈现「与她命运相关的事」，由设计宪法规定（玩家身份=祖魂之声、命运收件箱策展管线、意外但合理的归因门控、回来的一分钟）。本文与宪法冲突处以宪法为准。

> 从「有边界的回合制战棋对局」演进为「无限 LLM agent 接入的持久化沙盘大世界」的完整方案。
> 本方案的核心信条：**不重写引擎，只升级"作用域"**。现有后端已经是一台「LLM 自主 agent + 全量审计留痕」引擎，大世界所需的原子能力（自主决策、留痕、调度、成本护栏、分层存储、实时推送）已全部具备。我们要做的是把它从 `session` 作用域提升到 `world → region → tick` 作用域。

---

## 0. 一句话愿景与核心循环

**一句话**：你不是指挥官，你是一缕**附身在某具肉身上的"玩家魂"**。你创建一个角色丢进一个所有人共享的持久世界，离线时它由 LLM 续命、自由地谋生/结盟/复仇/相爱/背叛，上线时你用自然语言重新介入它的命运——而它的一生，每一笔都被忠实地写进可回放、可分享、可作呈堂证供的编年史。

**核心循环（玩家侧）**：
```
选模版创建角色 → 丢入大世界 → 角色自治发展(留痕) 
      ↑                                    ↓
   血脉传承 ←── 死亡/衰老              玩家上线接管
      ↑                                    ↓
   读编年史 ←── 处理"待决策收件箱" ←── 四种接管模式介入
```

**核心循环（系统侧）**：
```
个体动机栈 → LLM决策 → 服从度判定 → status.Mutator落地 → events留痕
      ↑                                                      ↓
  回灌个体记忆 ←──── Region聚合(势力兴衰/大事) ←──────── 区域事件流
```

**北极星指标**：D30 留存（养成型沙盘的生命线）。次级：付费率 4–6%、单角色被玩家"牵挂"的平均在世真实天数。

## 1. 设计支柱（七条不可动摇的原则）

1. **复用而非重写**：`status.Mutator` / `directive` / `ATB` / `ai.batch` / `预算护栏` / `dbdialect` / `WS hub` 全部复用扩展。只新增两块新代码：**作用域抽象（world/region/tick）** 与 **调度层（runner + worker + queue）**——它们是现有同步循环的"并发化包装"。
2. **决策用 LLM，结算用代码**（核心架构原则，详见 §1.5）：LLM 只负责"意图层"（这个角色**想做什么**）；所有"结算层"（战斗命中/伤害、移动、饥饿、经济产出、状态变更、零和裁决）一律由**数值驱动的确定性代码**计算，**绝不让 LLM 逐步算数**。一次"打架"由 LLM 决定"打"，由代码按数值把整场打完。
3. **留痕即产品**：现有 `events` 表（每次状态变更经 Mutator 强制产出可审计事件）不只是后台审计，而是编年史、纠纷仲裁、世界考古、传记书的唯一事实源。**留痕不可被金钱篡改**。
4. **公平靠"结算确定性"，不靠"不卖数值"这句话**（修正，详见 §11.3）：原"不卖 HP/伤害=不 P2W"的论证站不住——在共享世界里"谁先反应/算得更深"本身就是优势。真正的公平红线是：**一切零和结果由确定性代码在统一节奏上裁决，付费改变不了任何一场竞争的胜负**，只能改变叙事密度与体验质感。
5. **降级但不停机**：任何成本/算力压力下，角色都"继续活着只是变笨"（走规则 fallback），F2P 角色永不因算力不足而死。
6. **角色有自己的意志（但有一致性旋钮）**：玩家化身默认高度服从，人格冲突时仍可抗命——这是叙事张力的来源；但**早期低牵挂阶段要高一致、低惊喜**，随牵挂建立才解锁更多自治（详见 §11.5）。
7. **先验证再开发**：核心循环的两个致命假设（玩家是否会牵挂一个自治 LLM 角色、是否接受它违背己意）必须在写调度/分片代码**之前**用最便宜的方式证伪（详见 §11.6）。

## 1.5 LLM 与代码的职责边界（核心架构原则）

> 一句话：**LLM 决定"做什么"（意图），代码决定"结果如何"（结算）。** 一旦意图确定（无论由 LLM 还是玩家给出），所有机械结算——战斗、移动、经济、状态变更、零和裁决——全部由数值驱动的确定性代码完成，**绝不逐步问 LLM**。

这条边界同时解决三件事：**成本**（LLM 调用从"每步"塌缩到"每个决策点一次"，见 §11.2）、**公平**（结算确定性 → 付费改不了已开打的战斗结果，见 §11.3）、**可靠**（确定性结算可复现、可审计、不幻觉）。现有引擎其实已经部分如此：`generateUnitDecision`(LLM) 只产出意图，`combat_roll.go` + `status.Mutator` + ATB 全是代码——本节只是把它**升格为强制原则并贯穿全局**。

### 三层执行模型

| 层 | 谁来做 | 频率/成本 | 负责什么 |
|---|---|---|---|
| **① 反射层 Reflex** | 纯代码规则，0 LLM | 极高频、近零成本 | 绝大多数日常 tick：继续当前任务、饿了就吃、HP 危急就逃、基础寻路、关系/记忆数值衰减。**这是默认层**，多数角色多数时间只走这里 |
| **② 决断层 Deliberation** | LLM，稀疏 | 低频、昂贵 | 仅在"关键决策节点"升级调用 LLM 产出**意图**：首次遭遇敌对、收到新 order、社交/恋爱/交易/结盟/背叛抉择、战略岔路、目标重估(reassessGoal)。输出是"想做什么 + 对谁 + 说什么"，**不是数值** |
| **③ 结算层 Resolution** | 纯代码 + FNV 确定性掷骰，0 LLM | 跟随动作量 | 把①或②选定的意图**完整算完**：战斗命中/伤害/暴击/防御、移动/寻路、饥饿/疲劳/天气/生产/经济产出、状态变更落地(Mutator)、关系四轴 delta、视野、怀孕/衰老/死亡判定、零和仲裁。**任何一步都不问 LLM** |

**举例（就是你说的战斗场景）**：角色 A 这一次 ATB 唤醒，决断层（或反射层）产出意图 `{action: attack, target: B}`——**这是唯一一次可能的 LLM 调用**。接下来整场交战（A 砍 B、命中判定、伤害、是否暴击、地形加成、B 是否倒地）**全部由 `combat_roll.go` + 数值 + FNV 种子算完**，写进 events。LLM 在这场战斗里**一次都不再被调用**。下一个决策点（"打得不顺，要不要撤？"）才可能再升级到 LLM——而且通常先由反射层规则（HP<阈值自动撤）兜底，只有真正的岔路才上 LLM。

### 边界清单（落地时照此切分）

- **永远走代码（结算层）**：战斗结算、伤害/命中/掷骰、移动/寻路、ATB 速度与行动顺序、饥饿/疲劳/天气、生产/采集/经济产出、贸易价格撮合、关系四轴数值变化、视野/迷雾、怀孕/生育/衰老/死亡阈值、状态变更(Mutator)、**零和裁决(仲裁 tick，见 §11.3)**、离线 lazy catch-up 推演、服从度概率(`rejectProbability` 本就是公式)。
- **才走 LLM（决断层）**：动作**意图**选择、玩家自然语言 directive 的意图解析、对话**内容**生成、反思/记忆摘要/编年史**叙事渲染**、目标重估。
- **判据**：如果一件事"有确定的数值规则就能算出唯一结果"，它属于结算层、**必须用代码**；只有"需要在模糊情境里做带性格/价值判断的选择，或生成自然语言"才上 LLM。

> 这条原则把"<2% 上 LLM"从"预算耗尽时的降级模式"扶正为**设计的常态**：结算层本来就不耗 LLM，反射层兜住绝大多数 tick，只有决断层稀疏地花钱。成本模型因此从"线性随角色数爆炸"变成"随**有意义的决策点**增长"。

## 2. 世界模型：world / region / tick

### 2.1 空间：分区域(Region)，不分服

- 世界 = 若干 **Region**（复用 `world.generator` 的六边形地图，每个 Region ≈ 64×64 子图）。一个 Region 概念上等价于今天的一个 `session.State`，但**去掉敌我阵营预设**，改为"按地理聚集的角色集合"。
- **同一份权威世界**：Region 不是分服，角色可跨 Region 迁徙。"无缝"靠三层实现：
  - **渲染层**：前端只订阅角色当前 Region + 视野内邻区边缘（WS 订阅键从 `sessionID` 改为 `regionID`）。
  - **调度层**：Region 按活跃度分 **Hot / Warm / Cold** 三级（见 2.3）。
  - **边界一致性**：跨 Region 的关系/通信走现有**信鸽(pigeon)**系统异步投递，不要求两区同 tick。

### 2.2 时间：真实时间 → 世界 tick 的固定映射

| 项 | 取值 | 说明 |
|---|---|---|
| 基准节奏(Hot) | **1 tick = 真实 6 分钟** | 1 真实小时 = 10 tick，1 真实天 = 240 tick |
| 记忆半衰期 | `tau≈120 tick` ≈ **真实 12 小时** | 与"每天上线一两次"天然契合，**无需改 `memory_store` 常数** |
| Warm 节奏 | tick 间隔拉到 30 分钟 | 启用"批量补算" |
| Cold 节奏 | 不 tick，按需水化 | 落 Postgres 冷存 |

现有 `turns.State` 的 deploy/execute 两阶段语义改变：不再是玩家轮流，而是**每个世界 tick 内跑一次"微执行"**——调度器对 Region 所有角色跑一遍 ATB（`buildExecutionOrderByATB` 原样复用），但**只有 ATB gauge 累积到阈值(100)的角色才在该 tick 唤醒**。低速角色多个 tick 才动一次——这天然把昂贵的 LLM 决策"摊薄"到时间轴上。

### 2.3 冷热分层（承载无限 agent 的关键）

| 层级 | 触发条件 | tick 节奏 | 决策方式 |
|---|---|---|---|
| **HOT** | 有玩家在线/在场 | 5–10s/tick | 同步或异步 LLM 全速 |
| **WARM** | 离线但区内有活动 | ~600s/tick + 事件驱动唤醒 | 多数走规则 fallback，仅关键节点上 LLM |
| **COLD** | 长期无交互 | 不 tick | 被访问时 lazy catch-up（规则推演，不补 LLM） |

## 3. 角色创建：模版系统

两种创建路径输出**同一个 `CharacterTemplate`**，在引擎下游完全同构（执行循环无需区分）：

```
CharacterTemplate{
  ArchetypeID, Origin(出身), Motive(核心动机), Wound(核心伤痕),
  TalentTags[], PersonalityBias(8轴偏置), StartRelations[],
  SeedMemories[], Ambition(野心向量6维), IsRandom, Seed
}
```

实现为 `BootstrapRecord(seed)` 之外包一层 `ApplyTemplate(record, tmpl, seed)`：人格偏置叠加到 `GeneratePersonality` 的 center 上，天赋写入既有 `Skills.Specialties`，开局记忆走 `storeMemoryAndSyncHighlights(source="origin_story")`。

### 3.1 指定模版（6–8 个手调原型）

每个原型 = 策划手填档案（人格偏置区间 + 2–3 天赋 + 一句核心欲望 + 一句核心伤痕 + 3–5 条开局记忆 + 开局关系钩子）。

| 原型 | 人格偏向 | 欲望 / 伤痕 | 机制挂钩 |
|---|---|---|---|
| 游侠 Ranger | 勇气↑谨慎↑社交↓ | 找到下一个值得守护的人 / 部落被屠只剩自己 | 侦察、潜行 |
| 贵族 Noble | 野心↑魅力↑正直↓ | 复辟家名 / 政变中失势 | 谈判、统御，开局与某商人 trust+3 |
| 商人 Merchant | 社交↑谨慎↑ | 建立横跨大陆的商路 | 直接挂钩 `trade` 系统 |
| 刺客 Assassin | 攻击↑稳定↓正直↓ | 杀光当年雇主 / 被灭口的幸存者 | 潜行、伏击，开局对某 NPC fear+rivalry |
| 隐士 Hermit | 社交↓智慧↑稳定↑ | 被世界遗忘地活着 | 医术、草药，低社交主动性 |
| 匠人 Artisan | 谨慎↑正直↑ | 打出一件传世兵器 | 挂钩生产经济 |
| 逃兵 Deserter | 忠诚↓稳定↓ | 证明自己不是懦夫 / 丢下战友 | 开局 morale 偏低 |
| 吟游者 Bard | 魅力↑社交↑ | 写下会被传唱的故事 | dialogue/记忆传播权重更高，天然是世界考古的信息节点 |

### 3.2 随机模版（确定性抽签 + LLM 缝合）

用与现有同源的 `FNV` 哈希派生四条子种子，从加权抽签表取样：**出身(≥20) × 动机(≥15) × 伤痕(≥15) × 天赋**（理论 ≥4500 基础形态）。

关键是**自洽性约束矩阵**：抽签带兼容性权重（0.2 禁忌 / 1.0 中性 / 3.0 强相关）——"修道院弃儿"抬高"求知/避世"、压低"复仇"。最后调一次 LLM（复用 `narrative.go` 的 `TaskBackstory` 全套管线）把离散标签缝成 160–220 字 biography，强制要求"四个标签必须互为因果"。**随机的是标签组合，连贯性由 LLM 在强约束下补全**，失败回退本地模板。

### 3.3 野心向量（自由发展的"北极星"）

新增 `Profile.Ambition = {power, vengeance, wealth, lineage, mastery, freedom}` 六维各 ∈[0,1]，**出生固化、终生最大漂移 ≤0.15**。它与 8 维 personality 共同进入决策 prompt，是角色一切自发行为的"引力源"。

## 4. 自由发展：三层动机栈

角色"自由发展" = LLM 每次决策被喂入一个**结构化动机栈**，而非脚本指派任务。这不是新增 AI，而是把 `generateUnitDecision` 的 prompt 上下文从"战场态势"扩展为"动机栈 + 态势"。

| 层 | 内容 | 实现 |
|---|---|---|
| **L1 生理护栏** | 饿了 / 士气低落 / 重伤 | 直接读 `Status`，复用 `hunger.go`/`combat_shake.go` 高压情绪覆盖 |
| **L2 当前目标** | 一句话短目标（≤60字） | 新增轻量 LLM 调用 `reassessGoal`，每 24 tick 或重大事件后触发，存为 `memoryCategoryKnowledge` 高显著度条目 |
| **L3 野心** | 六维野心向量 | 长期不变，作为"目标合理性的引力" |

**涌现示例**：vengeance 高 + 记忆里有高情绪权重的"被羞辱"条目 → 目标自然漂向"接近并削弱仇人"；power 高 + 发现某首领 trust/fear 都低 → 漂向"取代他"；玩家 doctrine 与角色 L2/L3 冲突且 Loyalty 低 → 现有 `resolveDirectiveCompliance` 判 `refused`——**这就是"叛变"的现成出口，无需新机制**。

### 4.1 涌现嵌套：个体 ↔ 宏观闭环

新增轻量**"区域聚合器"**（每 Hot Region 每 24 tick 跑一次，纯规则 + 一次廉价 LLM 摘要）：扫描区内近期 events，统计各"势力"（= trust/affection 高且共享 doctrine 的角色**自组织聚类**，不预设阵营）的人口/财富/战损/领地。这份摘要：
1. 写入每个在场角色的 `memoryCategorySpatial` 作为"天下大势"→ 个体决策开始考虑宏观（"本势力在衰退，该跳船了"→叛变涌现）；
2. 触发宏观事件（势力崩溃/扩张意图）→ 再下放为具体角色的 L2 目标；
3. 喂给玩家的"世界态势面板"。

**宏观层严格只读不写**——所有状态唯一真相仍是 `unit.Status`，必经 `Mutator`。避免双写不一致。

### 4.2 生命循环：衰老 / 死亡 / 传承

现有 `ApplyFatalDamage → LivesRemaining → LifeStateDead` 是战斗致死路径，补两条非战斗路径（都走 Mutator + 新 reason-code）：
- **衰老**：`Identity.Age` 随 tick 增长（每 240 tick ≈ 真实 1 天 → +0.2 岁，压缩比可配，天然寿命约真实数十天到数月），过高龄阈值后每 tick 递增自然死亡掷骰（FNV 确定性随机）。
- **饥荒/累积创伤**：`StarvationTurns`/永久 `Injuries` 累积触发死亡。

**死亡 = 留存钩子而非终点**：写 `character_died` → 生成传记尾段（`narrative.go`/`legacy_hall.go`）→ 归档名人堂（冷存）→ **血脉传承**：遗产 + 未竟 `LifeGoal` 传给继承人，并把"父辈遗志"作为最高 importance 永久记忆注入（复用 `loot_inheritor.go` 扩展为全资产 + 记忆继承）。玩家化身死亡 → 选择**无缝接管一个血脉后代**（带情感记忆延续故事）或重新出生。

## 5. 离线自治 + 上线接管（最关键的差异化玩法）

**双轨体系**：离线不是暂停，而是角色按一份"离线宪章"在受预算约束的低频自治 tick 里继续活着；上线得到一个"收件箱 + 编年史"，用四种接管模式重新介入。**离线自治与在线接管共用同一决策器**（`generateUnitDecision → resolveDirectiveCompliance → Mutator`），行为/留痕一致。

### 5.1 离线宪章 Offline Charter

不发明新概念，直接实现为 Directive 的一个新 `Scope` 字段（`scope="offline_charter"`，旧 directive 默认 `scope="live"` 向后兼容）。自然语言经 `parseDirectiveIntent` 归一化，分三段：
1. **长期目标**（grow / explore / befriend / accumulate / defend）；
2. **红线禁令**（不许单挑、不许离开X区、HP<阈值必撤、不许卖传家宝）；
3. **社交授权**（可否自由恋爱/结盟/生育）。

未设置时给安全默认值（复用现有兜底，改写为"稳妥发展，遇重大抉择则冻结等待指挥官"）。

### 5.2 自治胆量曲线：越久越自主，但越保守

两条**独立可调曲线**解决这对张力（它们不矛盾，分别作用于决策的"广度"与"风险偏好"）：

- **广度 Autonomy(t)**↑：离线越久解锁越多可自决领域。`0–6h`={move,gather,eat,dialogue,defend}；`6–24h`=+{trade≤200金,produce,build,befriend}；`>24h`=+{ally,romance,migrate}。
- **风险 Risk-tolerance(t)**↓：`offlineCaution = 1 + 0.4·log2(1+offlineHours)`，乘进 `obedience.go` 的 `rejectProbability`（上限仍夹 0.85），离线越久越倾向把激进动作判为不执行。

直觉：**长期没人管的角色学会了"自己拿小主意、但绝不拿命赌大的"**。玩家可在宪章里显式设为"激进派/稳健派"。

### 5.3 冻结清单 Freeze List

| 自动做 | 冻结等定夺（进收件箱） |
|---|---|
| 移动、采集/生产、进食、日常对话、记忆/反思、低额交易、防御性撤退、关系自然演化 | 死战单挑、迁出宪章指定区域、卖/赠 `pinned` 重要物品、生育、结盟/背叛盟友、超出当前 autonomy 广度的领域、触发红线的诱因 |

实现：tick 决策器若产出落在冻结集的 action，**不落地**，而是经 Mutator 生成 `PENDING_DECISION` 事件入队收件箱，本 tick 改走安全替代（通常 Hold）。复用 `obedience` 的"refused→转 Hold"模式，触发源从"抗命"变成"权限不足，上交决策"。

### 5.4 低耗叙事模式（超长离线硬控成本）

离线 >24h 或预算护栏触发时，不再每 tick 调 LLM，而是每"叙事窗口"（1 世界日）调一次低成本 summary-LLM，把 N 个 tick 压成一段"这段时间我做了什么"（JSON Schema 强校验，复用 `memory_store.go` 的 `generateMemory2Compaction` 整套模式），再落成若干审计事件 + 一条高 importance 记忆。**token 成本下降一个量级**。护栏耗尽则整段走确定性规则推演，**F2P 角色离线永不"饿死在算力不足上"**。

### 5.5 四种接管模式

| 模式 | 操作循环 | 适用 | 付费关系 |
|---|---|---|---|
| **① 指令模式** | 下自然语言 doctrine/task/order → LLM 归一化 → 下个 tick 生效 | 轻度（2–5分钟，拨方向就走） | **免费主玩法，F2P 公平底线** |
| **② 附身/直控** | 该角色进"局部指挥时停"（复用 `ExecutionInProgress` 局部化），玩家逐步指定具体 action，包装成 `forcedOrder` 注入（复用 `activeImmediateOrderForUnit` 放行路径） | 关键战斗、精细走位、高光时刻 | 消耗"指挥专注"，每角色每日免费时长上限（防止把自主 agent 玩成传统手操战棋而架空 LLM 卖点） |
| **③ Copilot 共同决策** | 冻结点触发"选项生成"：LLM 以角色视角生成 2–3 个候选 + 各自理由/风险分/后果预测，玩家选或改写 | 中度（当军师不当保姆） | 基础 2 选项+1 免费重roll；第 3 个"激进选项"/额外重roll 为增值 |
| **④ 复盘模式** | 看离线编年史时间线 → 对每个事件做①追认②纠偏③补救 | 回来吃瓜+收拾摊子（**D7+ 留存钩子，非 D2 钩子**，见 §11.6） | 免费看全部；"反事实推演"为增值 |

**复盘对不可逆既成事件不能撤销**（尊重世界权威与留痕不可篡改），只能改变"未来"和"记忆解读"——这本身就是好叙事。

### 5.6 上线第一分钟：收件箱 + 编年史 + 状态卡

落地页三栏，目标 **60 秒内**让玩家"知道发生了什么 + 现在要做什么"：
- **状态卡（顶部一行）**：`离线 14h 3m｜阿采还活着,HP78,在北岭林地｜2 件待决策,1 件紧急`。
- **待决策收件箱（左侧·核心）**：按 `优先级=紧急度×不可逆度×情绪强度` 排序，每项带 Copilot 选项入口 + 倒计时（过期角色按宪章兜底，**绝不卡死世界**）。支持**批量处理**（全部追认/全部按角色倾向），照顾低频玩家。
- **编年史（右侧·叙事）**：离线期 top-N 大事，每条一句 LLM 旁白 + 可展开原始 events 链。一键"让阿采给我讲讲"= 调一次 `reflection` 第一人称回顾。

**收件箱不是新表**：`PENDING_DECISION` / `DECISION_RESOLVED` 作为新 reason-code，经 Mutator 落 events；收件箱 = "查询该单位未 resolved 的 PENDING_DECISION 事件"。天然继承留痕的可审计性与断线重连。

### 5.7 接管如何反向塑造角色（"你的角色但有自己的意志"）

玩家干预经**三条已有通路**反向塑造角色，而非直接覆写：
1. **记忆**：每次接管写 `source="player_intervention"` 记忆，importanceBoost 分级（追认+1、纠偏+2、强行违背意志直控+3 且情绪偏负）。
2. **性格漂移**：`Personality` **非**受保护字段，可经带 `PERSONALITY_DRIFT` reason-code 的小步调节器调整（每次 ≤0.03，单日单维 ≤0.10），保留个性惯性。
3. **忠诚闭环**：`Loyalty` 是受保护字段，只能经 Mutator——追认给 `LOYALTY_GAIN`、强推给 `LOYALTY_STRAIN`，直接进 `loyaltyRejectModifier`。**"你怎么对她，决定她以后多听你的"**——反复强推会推高未来 `rejectProbability`，形成"越按越不听"的负反馈，逼玩家尊重角色。

## 6. 留痕 → 编年史（把审计基础设施变成核心叙事功能）

> ⚠️ **形态修正（详见 §11.4）**：同类游戏证据表明**绝大多数玩家不会读长篇编年史**（长文通读率乐观也只有个位数~15%）。留痕的真实价值在**脑补燃料 / 社交货币 / 禀赋效应 / 仲裁工具**，不在"被通读的文本"。因此产品重心必须从"右侧栏长文叙事"转向**可一瞥的高光卡 / 推送 / 可分享图**；本章描述的四层叙事仍成立，但**长文改为按需懒渲染**（只为那 1% 考据党和仲裁场景花钱），LLM 预算大头投向高光卡。下文请配合 §11.4 阅读。

**绝不写第二套事实记录**：`events` 表（reason-code + importance）+ `RawEventLog` 是唯一事实源。新增 `chronicle_entries` 是**可随时从 events 无损重建的物化视图/缓存**，保存 `source_event_ids` 证据链。

### 6.1 四层叙事聚合

复用记忆系统的 importance(5–10) 与 salience 衰减做"值得写进史书吗"的筛子：

| 层 | 内容 | 数据源 |
|---|---|---|
| **角色【编年史/日记】** | 单角色第一人称/编年体 | 按 actor/target 过滤 events，复用 `battleReport` 第一人称模式 |
| **关系【纠葛史】** | 谁先示好、谁先背叛、何时反目 | `relation.notes_json`（已记录每次四轴 delta + reason + turn） |
| **势力【史书】** | 势力兴衰编年 | 聚合势力单位高 importance 事件 + 战报 |
| **世界【大事记】** | 阵营覆灭、大战、名人陨落 | 跨势力 importance≥9 事件 |

渲染**异步**（挂在阶段边界快照后，复用 `refreshUnitNarrativesAsync` 后台模式），**批量去重**（`GenerateJSONBatch`），受独立成本子预算（建议占 session LLM 预算 15%），超限降级为 `RawEventEntry.Summary` 模板拼接。

### 6.2 忠于事件：防 LLM 加戏

- **强约束**：渲染 prompt 只喂结构化 event 列表，system prompt 明确"你是史官，只能重组润色给定事件，不得新增任何未在列表中的人物/地点/因果/数字"。
- **幻觉回扫**（本地零成本）：抽取渲染文本的专有名词/数字，**必须是输入事件的子集**，否则判幻觉、回退规则模板（0 容忍）。
- **证据链**：每段 chronicle 存 `source_event_ids`，前端"展开证据"可逐条对照原始 reason-code 事件。

### 6.3 不可靠叙事 = 战争迷雾式历史认知

`chronicle_entries` 带 `perspective_unit_id`：同一组底层 events 对不同观察者渲染出**不同版本的历史**。复用现有 fog-of-war 可见性（`unitInVisionOfActor`）+ 信鸽 + dialogue 转述决定每个视角能看到哪些事件。于是背叛者写"我别无选择"，受害者写"他冷血地抛弃了我"，旁观者只记得"那天他们决裂了，原因不明"。转述经手的事件标"据说/传闻"并随 salience 衰减而失真——**"历史是被书写的"成为可玩机制**。

> 仲裁/举报场景**强制使用客观流水视图**（直接列 reason-code 事件 + 时间戳，无修辞），主观史仅作展示。

### 6.4 产品价值落地

1. **可回放的一生**：events 带 turn/时间戳，前端时间轴 scrubber 拖动重放编年史 + 当时关系四轴 + 记忆热度 + 配图。
2. **可分享传记**：死亡/主动立传时调 `legacy_hall` 的 `generateHallBiography`，渲染图文卡，归档 Postgres 冷存 + 短链分享。
3. **世界考古**：后来者的 LLM 决策上下文可查冷存名人档案（`injectHallMemoriesForUnit` 已实现），新角色"听说过"几百回合前的传奇——历史人物成为活世界的文化层。
4. **纠纷仲裁证据链**：relation 每次 delta 带 turn + `RELATION_BETRAYAL` reason-code，"谁先背叛谁"有不可争辩的时间戳证据链。

### 6.5 隐私

`chronicle_entries` 注册为 `privacy.go` 的 `EraseSessionPrivateData`/`PurgeExpiredSessionData` 下游级联对象。被遗忘权分级：本人日记可直接删；涉他人的条目走**匿名化中间档**（姓名替换为"一位旅人"，事件骨架保留）——既满足隐私又不撕裂世界历史。

## 7. 商业化：把 LLM 算力做成可感知资源

**核心主张**：把 `llm_budget.go` 隐藏的"每会话 $3 硬上限 / $2.85 护栏地板"升级为玩家可见的游戏资源 **天命点(Mana of Fate)**。每角色每天有一份天命预算，所有 LLM tick/对话/反思按 `metrics.go` 的真实 token 单价扣点。

> 换算：**1 天命 = $0.001 真实成本**。扣到护栏地板时角色不"凭空降智"，而是显式进入剧情化的**"灵感枯竭/休整"**状态（复用 `budgetGuardrailDecision`），前端提示"今日天命将尽，明日恢复或充能"。

### 7.1 免费档 vs 付费档（四个杠杆，全部复用现有系统）

| 杠杆 | 免费档 | 付费档 | 复用 |
|---|---|---|---|
| 模型档位 | DeepSeek（~$0.0005/tick） | OpenAI（~$0.0016/tick，3.2×） | `ai.providerOrder` / `TaskProfile.Primary` |
| 思考深度 | MaxTokens=400, effort=low | MaxTokens=900, effort=medium | `CompletionRequest.MaxTokens` / `ReasoningEffort` |
| 离线 tick 频率 | 30 分钟/tick（48/天） | 6 分钟/tick（240/天） | 离线 tick 调度 |
| 反应速度 | 应激信号合批进下一 tick | 应激可插队即时反应 tick | `reaction_queue.go` |

**体验差异**：付费角色"更像个活人"——记得更久、说话更长、反应更快、离线更密集；免费角色**依然是完整可玩的真 LLM agent**，只是节奏更慢、表达更短。

> ⚠️ **公平性修正（详见 §11.3）**：上表后两个杠杆（**⑤离线 tick 频率 / 反应速度**）在共享世界的**零和竞争节点**（贸易抢单、伏击、夺权、求偶）会直接转化成胜负优势——红队测算纯频率差就让付费方零和胜率达 ~83%，这就是 P2W。**纠正方案**：所有零和结果改由确定性代码在**统一节奏的"仲裁 tick"**上裁决（付费的高频只决定"多早把意图排进队列"，不决定谁赢）；并给免费角色在关键节点"决策质量地板"。**只有做完这层解耦，频率/反应类付费才允许上线**；在此之前，干净的付费主轴是 ⑦⑧⑩⑪（外观/冠名/留痕面板/名人堂）这些非竞争维度。

### 7.2 氪金点清单（按类，含红线）

| 类 | 氪金点 | 卖什么 | 公平性 |
|---|---|---|---|
| 算力 | ① 天命月卡（订阅） | 每日算力配额 + OpenAI 档 + 高频 tick | ✅ 不动数值 |
| 算力 | ② 天命加量包（一次性） | 临时翻倍今日配额 | ✅ |
| 槽位 | ③ 额外角色槽 | 同时养多个故事线 | ✅ 成本随真实 token 线性，定价覆盖 |
| 槽位 | ④ 模版图鉴扩展 | 解锁更多随机模版稀有词条池 | ✅ 只换风味 |
| 时间 | ⑤ 离线托管加速 | 临时拉满离线 tick 频率 | ✅ |
| 时间 | ⑥ 停机结算加速 | 跳过 downtime 等待 | ✅ |
| 外观 | ⑦ 立绘/皮肤/称号/信鸽样式 | 纯 cosmetic | ✅ 最干净 |

**🚫 必须避免的红线**（已按 §11.3 红队修正）：
- **根红线**：任何付费点若能在**两个角色争夺同一排他结果时改变胜负**，即判脏，必须经"仲裁 tick"解耦或下线。这取代旧的"不卖数值=合规"论证。
- **不卖任何战力**（HP/伤害/掉率/服从度 buff）——受保护字段本就禁止直接改写；
- **⑤离线托管加速 / 反应速度插队**属于"待解耦"项：上线前必须做到"高频只决定排队位序、不决定零和胜负"，且**插队封顶必须做在账号层**（防多角色槽绕过）；
- 不卖"让别人角色变笨/降速"的攻击性道具；
- 抽卡明示真实概率 + 十连保底，**稀有度只换故事风味不换数值上限**；
- 改命/复活**不可篡改既有 events 留痕**（否则破坏"留痕可信"这一产品核心）；
- **不分池/不分服**：付费/免费分世界会摧毁"单一权威共享世界"卖点，且只是把 P2W 藏起来（详见 §11.3）。

| 模式 | 定价 | 说明 |
|---|---|---|
| **订阅/算力** | 天命月卡 ¥30/月、季卡 ¥78；加量包 ¥6/¥30/¥68 = +300/+1800/+4500 天命 | 营收 ≥50%，健康可预测 |

### 7.4 防氪金反噬

- **算力硬护栏永不取消**：天命给护栏加"可充值上限"，但单角色单日仍有绝对硬顶（如 $1.2/天 = 1200 天命），超过强制 fallback——**任何付费都不会让单角色无限烧 token**。账号级再设日充值上限 6000 天命。
- **世界公平隔离**：战斗**结算**走 `combat_roll` + `FNV` 确定性随机对所有角色一致——付费改不了任何一场已开打的战斗结果（这是 §1.5"结算用代码"原则的直接红利）。**但要警惕**：付费角色"行动更频繁、反应更快"在**零和决策时点**（先到先得、先手伏击）仍是优势，这一层必须靠 §11.3 的"仲裁 tick"解耦，光靠"结算确定性"不够。
- **免费体验保护**：护栏触发包装成剧情化"灵感枯竭"而非"变蠢"，避免羞辱感；免费档全程真 LLM。

## 8. 技术架构落地（基于现有 Go 后端的演进）

最大结构性债务是 `single_player_sessions.state_json` 这块**整块 JSON State**——它把指令/对话/决策痕迹/LLM 交互/事件日志/单位 ID 列表全塞进一行，大世界下会变成 GB 级热点行，**必须先拆**。

### 8.1 数据模型演进

1. **引入 world/region/tick 三层骨架**：新增 `worlds` / `regions` / `world_ticks` 三张轻表（走 `dbdialect`）。复用 `world_tiles` 表里**已存在但闲置的 `region_id` 列**作为 tile→region 归属。tick 改为 per-region 逻辑时钟，deploy/execute 两阶段（`turns.State`）零改动。
2. **拆解巨型 State**：`RegionState` 只保留"当前活跃工作集"（目标单行 <256KB）；把 12 个历史切片（DirectiveHistory/Dialogue/DecisionTraces/LLMInteractions/...）**外移到旁路表**（按 region_id+tick 索引）；`RawEventLog` 并入 `events` 表（`session_id` 语义改 `region_id`，加 `world_id`/`tick` 两列）。
3. **单位归属迁移**：`units` 表加 `world_id`/`region_id`/`life_state`/`last_active_tick` 列；`status.Mutator.Apply` 写 events 时把 `record.SessionID` 换成 `record.RegionID` 即可，**主体逻辑零改动**。`ListActiveByRegion` 只 SELECT status 列做"轻量加载"。

### 8.2 Agent 调度

1. **三层活跃度 + 事件驱动唤醒**（见 2.3）：唤醒用 `agent_wake_queue` 表 + events 触发器（复用 `reaction_queue.go` 入队思路，作用域 session→region）。
2. **按 region 分片 + worker 池**：每个 region 由唯一一个 `region-runner` goroutine 持"区锁"（复用 `asyncExecutionRegistry` 的 `map[regionID]struct{}` 单飞模式）。runner 不直接调 LLM，而是把决策打包成 `decisionTask` 投入分层队列（内存 channel 做 HOT，`agent_decision_jobs` 表做 WARM/COLD 可重启恢复），由 `decision-worker` 池消费，按 region 批量聚合调 `ai.GenerateJSONBatch`。**两级背压**（batch 信号量 4–16 + worker 池全局 64）防 LLM 雪崩。
3. **离线混合推演**：WARM/COLD 默认规则推演，只有"关键节点"升级 LLM。lazy catch-up 用 `FNV(worldID+regionID+tick+unitID)` 确定性随机，每步仍经 Mutator 写 events——**"离线发生了什么"对玩家完全可见**。

### 8.3 LLM 成本与吞吐

1. **每用户/每角色算力账户**取代每局护栏：新增 `llm_budget_accounts` + `llm_charges` 流水（流水即"算力消费留痕"）。在 `ai.Service.GenerateJSON` 入口前加 `BudgetGate`，按 `character→user→region→world` 逐级扣减，耗尽即降级现有 `budgetGuardrail*` fallback（**零新增降级逻辑**）。
2. **分级模型路由 + 决策缓存 + 群体规则/关键节点 LLM 混合**：`ConfiguredTaskProfilesByTier(tier)` 按账户档位选 provider；决策缓存扩展 `ai.batch` fingerprint 思路做跨 tick LRU；**关键节点 gating**（HP<30%/新 order/首次遭遇敌对/跨阵营/贸易恋爱生育/玩家在场）才上 LLM。**目标：1 万 agent 的 region 单 tick 真实 LLM 调用 <2%（<200 次）**。

### 8.4 留痕存储规模化

`events` 热/温/冷三层（dbdialect 透明）：HOT 主库近 N tick（复合索引 `(world_id,region_id,tick)`）；WARM `events_archive`/分区表；COLD Postgres 冷存 + 可选对象存储（超老 events 打包 JSONL 落 S3/OSS，DB 留 manifest 指针）。`chronicle-compactor` 后台 goroutine（可由 `CronCreate` 调度）滚动迁移。编年史**按需懒渲染**并写 `chronicle_cache`。

### 8.5 实时性与一致性

- **单 region 单写者**：region-runner 是该区唯一状态写入者（持区锁串行 tick）。玩家 order 不直接改状态，而是入 region 的 inbound 指令队列，由 runner 在下个 deploy 阶段统一消费——**天然消解"两个玩家同时交互同一目标"的冲突**（按 ATB 确定性顺序裁决先后，败者收"目标已被占用"事件）。
- **WS 双层订阅**：`hub.sessionSubscribers` 升级为 region 频道 + character 频道；`BroadcastSessionEvent` 泛化为 `BroadcastChannelEvent(channel, ...)`，channel ∈ {`region:<id>`, `char:<id>`}。
- **跨 region 迁徙**：两阶段交接（源区标记 leaving + 冻结 → 目标区接管 + 写 migration 事件），跨区异步消息走信鸽 + `world.current_tick` 对齐。

### 8.6 迁移路线图（复用优先，按热点重构）

| 阶段 | 时长 | 内容 | 风险/收益 |
|---|---|---|---|
| **阶段 0** | 1–2 周 | **单 region == 单 session**，新增 worlds/regions 表只做"对局编目"，引擎不动，验证 WS region 订阅 + events 按 region 查询 | 零重构，先跑通 |
| **阶段 1** | 2–4 周 | **拆 State**：12 切片外移旁路表 + RawEventLog 并入 events；RegionState 瘦身；units 加 4 列双写灰度 | **收益最大**（消除巨型 JSON 行）；用双写+影子校验防不一致 |
| **阶段 2** | 3–5 周 | **异步调度**：region-runner + decision-worker 池 + wake/job 队列表，HOT/WARM/COLD 分层，离线 lazy catch-up；**主存切 MySQL**（dbdialect 已就绪，砍掉 SQLite 单连接限制） | 吞吐核心 |
| **阶段 3** | 持续 | **成本与留痕规模化**：算力账户 + 分 tier 路由 + 决策缓存 + events 三层归档 + 编年史按需渲染 | 商业化 + 规模化 |

**取舍铁律**：`status.Mutator` / `events` / `directive` / `ATB` / `ai.batch` / `budget fallback` / `WS hub` / `dbdialect` 全部复用扩展，**绝不重写**；只新增"作用域抽象"与"调度层"两块新代码。

### 8.7 新增表/字段清单（速查）

- **新表**：`worlds`、`regions`、`world_ticks`、`directives`、`dialogue_messages`、`decision_traces`、`llm_interactions`、`battle_reports`、`moderation_reports`、`agent_decision_jobs`、`agent_wake_queue`、`llm_budget_accounts`、`llm_charges`、`chronicle_entries`/`chronicle_cache`、`chronicle_archive`。
- **改列**：`events` 加 `world_id`/`tick`；`units` 加 `world_id`/`region_id`/`life_state`/`last_active_tick`；`directive` 加 `scope`；`inventory item` 加 `pinned`。
- **新 reason-code**：`character_born`、`character_died`、`vengeance_fulfilled`、`faction_collapse`、`PENDING_DECISION`、`DECISION_RESOLVED`、`PERSONALITY_DRIFT`、`LOYALTY_GAIN`、`LOYALTY_STRAIN`。
- **新接口**：`unit.Repository.ListActiveByRegion/CountActiveByRegion/SetLifeState/TouchLastActiveTick`；`ai.Service.BudgetGate` + `ConfiguredTaskProfilesByTier`。

## 9. 统一风险登记册

| 风险 | 缓解 |
|---|---|
| **无限 agent LLM 成本爆炸** | 三道闸门叠加：ATB gauge 唤醒制（多数角色多 tick 才决策）+ 冷热分层（Warm/Cold 走 fallback）+ 账户级 BudgetGate；混合 gating 让真实 LLM 调用 <2%；worker 池全局在途上限 64 + batch 背压 |
| **离线角色做出"灾难性"后果致玩家挫败** | 冻结清单把所有高代价不可逆动作上交收件箱；pinned 物品永不自动卖；离线"谨慎模式"；死亡仅走衰老/被动卷入不主动送死；复盘可对未来纠偏 |
| **直控太好用架空 LLM 卖点** | 直控消耗"指挥专注"+每日免费时长上限；违背意志直控扣 Loyalty + 写负面记忆 + 推高 rejectProbability（越按越不听） |
| **接管把角色个性碾平 / 干预无效** | 性格漂移每次≤0.03、单日单维≤0.10（保留惯性）；Loyalty 反馈即时可感（收件箱回执直说原因 + 闪回记忆 ID）——慢漂移 + 快反馈分层 |
| **编年史 LLM 加戏破坏留痕可信** | 叙事永不写入事实源；本地 NER/数字回扫（0 容忍超集即回退）；仲裁始终以原始 reason-code 事件为准 |
| **巨型 JSON 拆表数据不一致** | 阶段 1 双写 + 影子校验 + 后台对账；session_phase_snapshots 作回滚锚点；影子流量验证一周再切 |
| **SQLite 单连接成吞吐瓶颈** | 阶段 2 切 MySQL（业务零改）；切换前靠"单 region 单写者"区锁在 SQLite 跑 MVP |
| **氪金演变 P2W**（红队确认：当前设计就是 P2W，见 §11.3） | 零和结果改由确定性代码在"仲裁 tick"统一裁决（付费只决定排队位序不决定胜负）+ 免费角色关键节点"决策质量地板"+ 反应频率**账号级**封顶 + 零和资源尽量 PvE/正和化；干净付费主轴转向外观/留痕/叙事密度 |
| **复活/改命滥用削弱叙事张力** | 复活每角色限 2 次且每次留痕；改命只改未来权重；用稀缺 + 留痕保留死亡/转变的戏剧重量 |

## 10. MVP 建议（先验证什么）

把"完整愿景"压成一个能验证**核心假设**的最小切片。核心假设是：**"离线时我的角色由 LLM 自由发展、上线后我介入它的命运"这个循环本身就好玩、且留存。**

> 🛑 **写 MVP 代码之前先过两道便宜的验证闸（详见 §11.6）**：① 假门测试（落地页测意愿，留资转化 >15% 才继续）；② 人肉后台/绿野仙踪（真人手写角色离线经历，测 D2 主动回访 >25% + 牵挂表达 >40%，并专门 A/B 测"角色违背玩家意志"两组反应）。**这两道闸验证的是两个致命假设——错了整个方向就崩，绝不能等 MVP 上线才发现。**

**MVP 范围**（约对应阶段 0 + 阶段 1 的一部分 + 玩法最小集）：
1. **单 region（== 单 session）持久化不结束**，世界 tick 调度器按真实时钟低频推进（先只做 WARM 节奏，不追求海量并发）。
2. **模版创建**：先做 4 个指定原型 + 随机模版（标签抽签 + LLM 缝合）。
3. **动机栈 L1+L3**（先不做 L2 reassessGoal 的高频版，降本），让角色能自发谋生/社交/移动。
4. **离线自治 + 离线宪章 + 冻结清单**：角色离线续命，高代价动作进收件箱。
5. **接管模式 ①指令 + ④复盘**（最便宜的两个），②③后置。
6. **D2 悬念钩子**（关键，见 §11.6）：每个新角色在玩家下线时刻必带一个"强情绪 + 不可逆 + 有倒计时"的待决策（战书/求婚/仇人现身），配 D2 推送"你的角色正在等你的决定，过期 TA 会自己选"。**这是 D2 回访引擎，不是编年史。**
7. **高光卡 + 上线第一分钟收件箱**（不是右侧长文，见 §11.4）：把离线大事渲染成 1–3 张可一瞥的高光卡；角色编年史长文按需懒渲染。
8. **天命点 + 免费/付费档骨架 + 月卡**：验证算力资源化的付费意愿；**频率/反应类付费暂不上线**（待仲裁 tick），先卖外观/留痕导出等干净维度。

**MVP 不做**：海量 region 分片、worker 池规模化、events 三层归档、抽卡/通行证、直控/Copilot、势力史/世界大事记。这些都在验证核心循环跑通后按阶段 2/3 加。

**MVP 成功标准**：①核心循环跑通且单角色 LLM 成本可控（护栏内）；② **D1→D2 自然回访 >20%（推送召回 >35%）、D2 收件箱处理率 >70%**；③ 按"叙事投入度"**分层**看留存（核心人群 vs 大众，别用平均数判生死）；④ **单活跃玩家角色日均真实 LLM 成本不吃穿目标 ARPU**（留存与成本必须一起活）；⑤留痕→编年史的"零撒谎"校验通过。

## 11. 设计审视与修正（六问的诚实回应）

> 本章是对六个尖锐质疑的诚实回应与对原方案的修正，由"容量数学 / P2W 红队 / 行为证据 / 留存验证"四路深度分析（多处实读后端代码）综合而成。前文若与本章冲突，**以本章为准**。

### 11.1 编年史的起源与归属：三层，"真历史"只能从真实事件累积

不是"系统全生成后玩家来影响"，也不是"从第一个玩家开始才有历史"——**NPC agent 本身就在写史**（无限 agent 自治是前提，零玩家在线世界也在演进）。准确说法是三层：

| 层 | 谁生成 | 保真/可审计 | 作用 |
|---|---|---|---|
| **神话/史前层** | 系统（程序化 + 轻量 LLM） | 低 / ❌ 不可审计、不可变背景板 | 解决冷启动：地名、远古恩怨、几个已入名人堂的"传奇先人"。复用 `injectHallMemoriesForUnit` 让新角色"听说过" |
| **开服前 burn-in 活史**（建议） | 纯 NPC agent 自治跑一段 | 高 / ✅ | 让世界开服即有鲜活近期历史，不是空场 |
| **降临后活史** | 所有 agent + 所有玩家共缔造 | 高 / ✅ 可仲裁 | 真正的活历史 |

**原则**：可审计的"真历史"不能凭空预生成（它就是 `events` 派生物，没有事件就没有历史）；但世界需要一层**不可审计的"传说背景"**解决冷启动。玩家是其中**能动性最高的特殊参与者**，不是唯一作者。
> 决策杠杆：**赛季制世界**（每季归档名人堂、重开新世界，从神话层 + burn-in 起步）对"公平起跑/新人进入"友好得多，且与 §11.3 的公平诉求协同——建议认真考虑。

### 11.2 引擎容量：原样撑约几百，瓶颈是 DB 不是 LLM

实读代码测算（1 万角色）：
- **LLM 成本可控**：三道闸门（ATB 唤醒制 + 冷热分层 + §1.5 的"<2% 上 LLM"）叠加后稳态约 **37 次 LLM/分钟**，免费档约 **$40/天**；**加 prompt 缓存降到约 $11/天**（DeepSeek 上下文缓存命中价是输入的 10%）。LLM 不是第一瓶颈。
- **真正先撑不住的是四笔工程债**（均有代码出处）：①巨型 `state_json` 单行（`repository.go:43`，GB 级热点行）；②连接硬顶（`SQLite SetMaxOpenConns(1)` 全串行 / `MySQL SetMaxOpenConns(8)`）；③单 region 单写者串行（玩家扎堆时一个 goroutine 串行整区）；④每决策约 **15 次 DB 写**（`Mutator.Apply` 每 mutation 做 `GetByID + Save + INSERT events`，一动作 5–8 个 mutation）。

**判定**：原样（单 session/SQLite/未拆 state）稳态**约几百活跃角色**；**"几千"**需阶段 1（拆 state）+ 阶段 2（切 MySQL + region 分片 + worker 池）；**"几万"**再加 prompt 缓存 + events 分区归档 + 验证"<2% 上 LLM"在真实玩家聚集下成立。

**立即可做的高 ROI 改动**（不用等大重构）：
1. **给 ~2800 token 的静态决策 prompt 加缓存**（成本和 TPM 各降 60–75%）——实读 `buildDecisionPrompt` 静态指令文本已达 2077 汉字 + ~1800 ASCII，每次全价重算。
2. **把 `Mutator` 的"每 mutation 1 Save + 1 INSERT"改成单决策批量事务提交**（15 次 DB 往返 → 2–3 次）。
3. **MySQL `SetMaxOpenConns(8)` 调到 64–128，并读/写/WS 分离连接池**。
4. **客户端侧 RPM/TPM 限速器 + 429 退避**——免费/代理端点 RPM 常只有 20–60，晚高峰先撞限流。

> **最大不确定（最该先压测）**：所有数字建立在"分层比例"假设上。真实留存玩家会在晚高峰**扎堆同一 region**，HOT 比例、LLM 触发率、单 region 串行压力、provider RPM **四个指标一起恶化**，37 次/分可能实际是 100–300 次/分。

### 11.3 公平性：当前设计就是 P2W —— 必须靠"仲裁 tick"结构性根治

**红队结论（已采纳）**："不卖 HP/伤害=不 P2W"是**定义作弊**。在涌现式社会里零和胜负由"谁先到/谁先反应/谁算得深"决定，而这恰好是要卖的四个杠杆。量化：纯 5× 频率差 → 零和竞速付费胜率 **≈83%**；叠加应激插队 → **90–95%**；业界 ≥70:30 即判碾压。代码自证：`combat_roll.go` 对付费档盲（数值公平守住了），但 `executor_loop.go` 的 `taskBiasSpeedBonus`（priority 10 → +4.3 速度抢先手）+ `reaction_queue.go` 的抢占机制，已经把"优先级/反应"换算成先手——方案要卖的"应激插队"正是给这条先手链路标价。

**真有效的结构性解法**（按重要性）：
1. **零和裁决与 tick 频率强制解耦（唯一根治）**：所有排他零和结果（成交/占席/伏击发起/求偶突破/政变）走**统一节奏的"仲裁 tick"**，用确定性代码（FNV + 双方投入/属性/距离，对应 §1.5 结算层）裁决。付费高频只让你"更早把意图排进队列"，**不决定谁赢**。复用引擎已有的"单 region 单写者 + ATB 确定性排序"，并用 `atbMomentumPenaltyFactor=0.85` 防连续抢占垄断。
2. **免费角色"关键节点决策质量地板"**：在零和关键节点强制临时升档到付费同级模型/token，边际成本由"<2% 上 LLM"吸收——把模型质量差锁死在"日常叙事密度"这种无害维度。
3. **应激插队全局封顶，且做在账号层**（否则多角色槽绕过）。
4. **降低零和烈度**（PvE/正和化）：市场深度撮合替代抢单、多人可并存的影响力层级替代唯一首领——没有零和就没有"赢"可买。但注意：**降太多会失去背叛/夺权/复仇的戏剧张力**（那正是留存钩子来源），需权衡。

**必须点名的自欺**：分池/分世界匹配——摧毁"单一权威共享世界"核心卖点，且只是把 P2W 藏起来（付费池内仍是高频碾压）。

**可证伪的红线判据**：任何付费点若能在两个角色争夺同一排他结果时改变胜负，即判脏，必须解耦或下线。商业化论证从"不卖数值=不 P2W"改为"**零和结果统一节奏裁决 + 付费只买叙事密度与体验质感**"。建议每个零和系统先做**蒙特卡洛模拟**（付费方设 5× 频率 + 插队 + 强模型，跑 10⁴ 次，付费胜率 >60% 即判 P2W），在接代码前就识别会碾压的系统。

### 11.4 编年史：绝大多数玩家不会读长文 —— 倒成"消费频率金字塔"

证据（RimWorld/CK3/Dwarf Fortress/90-9-1）：价值在**转述与脑补**，不在阅读生成文本；长文主动通读率乐观也只有个位数~15%。真实价值 = **脑补燃料 + 社交货币 + 沉没成本/禀赋效应 + 仲裁工具**。

**改造为三层金字塔，按消费频率倒置投入**：
- **顶层（覆盖 99%）= 可一瞥的高光卡 / 推送**：上线第一屏是 1–3 张"你不在时发生的大事"卡（3 秒读完，一句 ≤30 字旁白 + 情绪色）。学 RimWorld/Wildermyth **留白**，把脑补空间留给玩家（还省 token，与 §6.2 防加戏一致）。
- **中层（少数人）= 可展开事件流时间线**。
- **底层（1% + 仲裁）= 完整证据链长文**，**纯按需懒渲染**。

把原 §6.1 建议的"15% LLM 预算投长文"**重分配约 2/3 到高光卡/分享图**。**死亡传记卡 = 最强分享触发点**（Spotify-Wrapped 式竖版图，对应禀赋效应峰值与付费传记导出）。把"多视角不可靠历史"（背叛者版 vs 受害者版）做成**社交对线话题**而非单人展示。新增指标：高光卡曝光率 D1 玩家 80%+ 瞥见 ≥1 张；长文展开率北极星只定 5–15%；D30 自发分享率 8–15%。

### 11.5 接管 vs 托管：同一决策引擎，不同输入与权限 + 一致性旋钮

**相同（刻意保持，是特性）**：决策管线完全一致（`generateUnitDecision → resolveDirectiveCompliance → Mutator → events`），动作空间/状态规则/留痕一致，使角色"前后是同一个人"。

**差异六处**：①输入权威（在线 order 高优先级+化身高 Loyalty近必从 vs 离线 charter 弱而笼统）；②自治广度/风险（离线有胆量曲线：越久越自主但越保守；在线无保守系数）；③谁拍板（在线你可授权冻结清单里的高代价动作；离线这些被冻结入收件箱）；④粒度（在线四模式，直控可 forcedOrder 绕过服从度；离线 LLM 自主选）；⑤保真/成本（离线降频/规则 fallback；在线全保真，付费档更高）；⑥反应速度。

**新增"一致性旋钮"**（应对 §11.6 暴露的风险）：托管与接管差异越大，新玩家越觉得"这不是我的角色"。**早期低牵挂阶段让托管高度贴合 charter、保守、少惊喜**，随牵挂建立才解锁更多自治。区分**好的不一致**（角色有个性）与**坏的不一致**（违背明确红线/出戏）——charter 红线是硬约束，红线内才允许人格发挥。

### 11.6 D2 留存：编年史是 D7+ 钩子不是 D2 钩子 —— D2 必须是"悬念决策"

**诚实结论：D2 留存是这个产品的生死线，而当前设计把 D2 钩子放错了地方。**
- D1→D2 时玩家只做了"选模版 + 设宪章"，**情感账户余额为 0**，故事再好也是"别人家孩子的事"。"复盘读编年史"是 **D7+ 钩子**。
- **D2 钩子必须是**：收件箱里**一条紧急 + 不可逆 + 有倒计时 + 关乎我角色命运的悬念决策**（战书/求婚/仇人现身/盟友将叛），用 D2 推送"你的角色阿采正在等你的决定，过期 TA 会自己选——而你可能不喜欢"。这叠加了养成牵挂 + 放置"不回来就错过" + SLG 损失厌恶。
- **当前设计亲手削弱了它**："过期角色按宪章兜底，绝不卡死世界"杀死了损失厌恶——需在"不卡死世界"与"保留真实错过感"间找平衡（兜底，但让玩家事后看到一个他不喜欢的结果）。

**两个致命假设（错了整个方向就崩）**：① 玩家会不会对一个"自己没怎么塑造、还会违背己意"的 LLM 角色产生足以驱动回访的牵挂？（与"角色高度自治"内在张力，无直接先例）② 玩家是否接受角色离线做他不认可的事？

**由便宜到贵的验证三步（写任何调度/分片代码之前）**：

| 步 | 做法 | 成本 | kill criteria |
|---|---|---|---|
| **① 假门测试** | 落地页讲清核心循环，测预约留资 + "最打动你的卖点"问卷 | 几千元、一周 | 转化 <8% 砍方向；8–15% 谨慎；**>15% 继续** |
| **② 绿野仙踪/人肉后台** | 招 20–30 种子用户，真人手写角色离线经历 + 收件箱 + 推送召回；**A/B 测"角色违背玩家意志"两组** | 人力非开发 | D2 主动回访 <25% 砍；牵挂表达 <40% 红灯；**违意组 >50% 表达愤怒/弃坑则"自治"方向需大改** |
| **③ 极简 MVP** | 见 §10，埋全 AARRR 漏斗 | 小开发 | 北极星 = **D2 收件箱处理率**（>70% 强成立）；按叙事投入度分层看留存 |

**留存基准锚点（设阈值用）**：放置类 D1 35–50%/D7 12–20%；SLG D1 30–40%/D7 15–25%；AI 陪伴类核心人群 D1 40–60%/D7 20–35%（但靠即时双向对话，本设计是延迟二手叙事，应保守对标下沿）。**很可能的真相**：核心人群（5–15%）很粘、高 LTV，大众 D2 即流失——这可以是好生意，但别按大众放置游戏的规模预期定目标和买量。
> 与 §11.2 合流的风险：**留存假设和成本假设必须一起活**。MVP 阶段同步实测单活跃玩家角色日均真实 LLM 成本，吃穿 ARPU（目标 ¥0.9–1.6/MAU）则商业模式照样崩。

### 11.7 修改优先级（一句话主线）

四个硬真相指向同一修正：**从"自治 + 长文 + 零和"转向"可控感 + 可炫耀的一瞥 + 公平的竞争层"。**

| 优先级 | 动作 | 对应 |
|---|---|---|
| 🔴 P0 | **先验证再开发**：假门→人肉后台→MVP，证伪"牵挂 LLM 角色""接受违意"两个致命假设 | §11.6 |
| 🔴 P0 | **落实 §1.5 边界**：决策用 LLM、结算全代码（这是成本与公平的共同地基） | §1.5 |

| 🟠 P1 | **编年史去长文化**：高光卡/推送/分享图为主，长文懒渲染，预算重分配 | §11.4 |
