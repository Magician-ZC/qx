## 一、已落地的确定性结算原语（`backend/internal/engine/`，纯逻辑、可测试）

「决策用 LLM、结算用代码」的地基。全部有测试。

| 包 | 能力 | 对应设计 |
|---|---|---|
| `engine/decision` | 三层决策路由（反射/决断/结算）`Router.Route` + **归因校验**`ValidateAttribution`（「意外但合理」代码强制）+ 门控意外`GateSurprise` | 宪法 §1.5 / §5 |
| `engine/arbitration` | 零和确定性裁决 `Resolve`（胜率∝Score、频率/顺序无关、付费不进、A-Res 加权抽样） | 宪法 §11.3 反 P2W |
| `engine/relevance` | 相关性 `Score`（relative-importance + noisy-OR）、`HopFidelity`、`StopPropagation`、`RouteFor`（命运三档）、`ConsentTierFor`、`MatchScore` | 事件耦合 M1 / 跨玩家 |
| `engine/encounter` | `ContributionScore`、`AllocateLoot`（贡献分赃走 arbitration）、`PenaltyCap/DegradePenalty`（后果分级闸）、`SoloAllowed` | PvE §5/§6/§7 |

| `villageseed` | 确定性「出生即 20 人关系网」`Generate`（FNV+splitmix64，含人格/种子记忆/秘密/人生目标/四轴关系）| 宪法 §4.5 |

## 四、贯穿的红线（已在代码/设计层守住）
- **付费不改命运胜负**：战斗/分赃/掉率走 `combat_roll`/`arbitration` 确定性、付费不进 Score；受保护字段禁直改（statuslint 守门）。
- **D0–D3 保护**：后果分级闸硬锁不可逆（`encounter.DegradePenalty` + elite 候选层=1，测试验证 lives 永不归零）。
- **留痕不可篡改可仲裁**：一切经 `status.Mutator` + reason-code，或 `EmitProcessEvent` append-only。
- **祖魂身份**：玩家影响处境不改灵魂；抗命经归因叙事化为「她在变成她自己」。
