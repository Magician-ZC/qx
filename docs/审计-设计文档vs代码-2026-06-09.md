## 2. 🔴 P0 — 结构性总开关（一个缺口压死一整层）

### 2.1 `state.WorldID` 恒空 → 跨玩家整层默认 no-op（最高杠杆）
- **现状**：真实建局/匹配/村庄路径一律传 `worldID=""`；`AssignSessionToWorld` 仅测试调用；`world.Create` 只有 ops HTTP 入口。
- **后果**：`surfaceCrossEventsAtBoundary` / `scanAndMatch`（撮合）/ `recordWorldizedKill` / 七交互 / consent / world boss / auto_match **全部默认 no-op**，且入口多在 `opsTokenGuard` / developer 门后 → 默认部署玩家零可达、零自治触发。
- **这是「事件耦合与跨玩家关联」问题B 整层的总开关**：修这一处（建局时把 session 分配/创建到一个世界），下游一大批 flag_gated/partial 跨玩家功能会一起点亮。

### 2.2 AARRR 埋点 write-only → 增长漏斗/北极星不可观测
- **现状**：`product_events` 富埋点只写不读（全仓 `SELECT product_events` 仅测试）；运营看板的「转化漏斗」实际只读 `fake_door_leads`（假门）；`EventReturnVisit` 常量定义但零发射点。
- **后果**：D2 organic 回访率、分享率/DAU、ARPPU 分层、首充/复购等北极星指标**全部算不出**；OpsDashboard 看到的是残缺口径。
- **缺**：`product_events` 读聚合端点 + 关键事件真实发射（return_visit/character_died/share 等带 user/session 维度）。

> P0 小结：这两条不是"再写个功能"，而是**让已建好的一大片能力真正通电**——杠杆远高于单点功能。

## 3. ❌ 真缺失模块（missing，17 项，从未实现）

**命运/叙事（设计宪法 + 事件耦合 M1）**
- **M1 data-driven 翻译层 `translation_templates`**：把宏观世界事件（战争征召/贸易欠债/势力投靠…）按条件翻译成个人命运 beat 的**数据驱动模板表 + 规则引擎**。当前世界事件→前台只有死亡/威胁/血仇/跨玩家/冻结几条**硬编码**路径，覆盖矩阵（reason_code × anchor_kind）缺失。
- **OOC_REJECTED 审计事件（§5.2）**：归因 OOC 回退当前只增进程级内存计数，无 reason-code、无持久审计行（重启即丢、无单条溯源）。
- **§8 乐趣度量三键反馈**：高光卡「意料之中 / 有点意外但合理 / 太离谱」三键轻交互 UI + 埋点 + 聚合——这是 GDD 核心乐趣指标「惊喜命中率 / OOC 率」的地基，前后端全无。

**跨玩家深水区（事件耦合 问题B 余项）**
- **双向世界化 Worldize**：我的角色行为→点亮他人锚（对 NPC 反查 + PROPAGATION_INBOUND 探针）。
- **NPC「有意义事件」预算**：anchor-weighted 生成预算 + 入向探针配额 + 防茧房破圈预算。
- **跨玩家零和裁决 §2.6**：争夺排他标的（同一联姻对象/继承席位/同批战利品）的 tick 统一结算窗口 + 离线宪章兜底投入 + 退而求其次补偿。

**PvE 深水区（PvE威胁系统 §5/§6/§9）**
- **副本 dungeon（异步分段推进）**：踏入态/segment 切分/按 tick lazy catch-up/floor boss FirstContact 暂停/HP 反射撤退抉择/charter 超时续命——整套缺（当前仅枚举占位）。
- **战利品传承 SoulBound/Legacy/继承**：`ItemStack` 的 SoulBound/Durability/IsLegacy 字段、epic 不可交易掉落、Clutch→Legacy 升级、死亡传承链（继承人+名人堂+下一代记忆）。
- **失败后果向旁观者传播（一人一版）**：`region_ravaged` cross_event 向「在乎她的人」按 relevance.Score+HopFidelity 扇出（WorldizeDeath 已对死亡做了类似，但未泛化到威胁失败）。

**产品/商业化（PRD）**
- **会员/权益→玩法 perk（叙事密度付费档）**：P2W 修正后的核心收入重定向**零实现**——买了会员游戏内毫无差别，entitlement 是死数据。
- **Live-Ops**：赛季重开/burn-in 活史/传承回流/GM 世界事件注入/A-B 平台全缺（PRD §8「无 live-ops 即慢性死亡」对应能力为空）。

**沙盘/忠诚**
- **忠诚闭环 LOYALTY_GAIN/STRAIN**：reason-code 已登记但**零 Mutator 调用方**；玩家接管（追认/纠偏/强推）→忠诚增减的「越按越不听」负反馈不存在（§5.7 未落地）。

**验证实验**
- **A/B 实验框架**：无 ab_bucket 分桶/分配/持久化、无按桶拆分的漏斗/留存查询 → 设计的红线 A/B、卖点 A/B/C、服从 vs 违背 A/B 在代码侧无法跑。

**其它（P3）**
- PvE 专属 ~24 个 reason-code（§9）未登记（当前留痕被通用码掩盖）；events 三层归档（温/冷 + chronicle-compactor）未建（规模化阶段3）。
