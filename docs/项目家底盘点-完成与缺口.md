## 一、一句话总评

**项目仍是「机制扎实、原语精良的 First Playable 原型」，但本会话把「命运开盒前台、战斗情绪深度、可靠性、安全合规与商业化地基」从缺口推进到了可用——核心乐趣闭环首次接进主客户端。**

相比上一版的最大变化：命运四槽 + 抗命溯源卡 + PvE 入口**已接进主指挥客户端**（不再只活在独立 `#fate`）；combat_shake 高压情绪覆盖**从死代码变为主循环实接**；技术债清理使 `go vet` / statuslint 转净（3→1）；内容安全、ops 端点鉴权、真实 IAP/实名网关、region 分片、商业化 billing 全部落地（多为 flag-gated 切片）。**仍未跨过的三道门**：① 后端能力的**前端 UI 覆盖**（治理/商业化/跨玩家仍无界面）；② **自治驱动**（PvE/跨玩家/撮合仍靠 ops-HTTP 手动触发，非玩法自动产生）；③ **账户级成本闭环**（配额闸是死代码，根因是 session 未绑 account_id）。

## 三、本会话相比上一版盘点的「净推进」

- 命运四槽（FatePanel）+ 抗命溯源卡（DefianceCard）**接进主指挥客户端 App.tsx**（顶栏「命运」按钮、活动流/决策摘要三处渲染抗命卡，防机器编码裸泄）——核心卖点首次对主战棋玩家可感知。
- combat_shake 高压情绪覆盖**接入决策主循环**（resolveExecution 调 resolveCombatShake，三路 Modifiers 乘性合流），并修 retreat（返回相邻格）/rage（用可见敌集）两处 load-bearing。
- 命运 **M2/M3 硬规则**：每日待决策≤3 防轰炸、48h 过期倒计时、过期兜底回响卡、后果分级闸（牵挂放大 urge 代价、不可逆类高牵挂拒绝自动兜底）。
- **技术债清零**：删 trade.go 死函数（消除 `go vet` unreachable）+ hunger 断粮致死走 Mutator（statuslint 3→1）；新增 ATB/combat_roll/记忆衰减确定性单测。
- **治理安全硬化**：内容安全双向审核接 3 chokepoint、4 个高危 ops 端点套 opsTokenGuard、真实实名 HTTPRealnameVerifier（不再信任客户端 bool、PII 不落库）。
- **商业化 + 大世界地基**：billing（SKU/购买/权益/真实 Apple+Google IAP 收据校验，未配网关安全拒绝）、region 分片接 runner 主循环（flag-gated 多实例租约）、regions/world_ticks 表 + 注册表、MySQL 连接池 8→64。
- AARRR 漏斗：**8 个事件已实际 Emit**（session_created/character_created/account_registered/purchase/intervention/decision_pending/inbox_opened/decision_resolved）。

- 确定性 combat_roll（FNV）、地形/天气/设施战斗修正、技能、战场生成；**本会话补 ATB/combat_roll/记忆衰减确定性单测**。

## 五、部分完成（有能力但未接主链路 / 默认 flag 关 / 半接入）

### 引擎原语「建好未接」（基本未变）
- **decision.Router 真短路默认关**（QUNXIANG_REFLEX_SHORTCIRCUIT），战斗主循环仍直调 LLM。
- **scheduler/encounter/attachment 全挂 region-runner**（默认关）；attachment 的 ReturnVisits 恒 0。
- **arbitration 唯一自治入口缺位**：MatchIntoSocialObject/boss 分赃仅 ops-HTTP，无自治玩法构造候选——反 P2W 机制保证尚无玩家可感知应用点。

### 命运层剩余
- **命运卡倒计时前端不渲染**（后端已返回 expires_at/countdown_hours，前端 FateCard 类型不透传）——M2② 只做了后端一半。
- **主指挥客户端的局不种村**：SeedVillage 只从 `/api/units/bootstrap` 触发，主客户端 `createSinglePlayerSession` 路径不调——主玩法局没有 20 人关系网。
- M3 后果闸未泛化：fate-resolve 路径 daysAlive=0、D0-D3 硬锁只在 threat 遭遇链生效；§4.3 provenance 链未在 fate 后果路径强制。

### 治理安全剩余
- **未成年宵禁/防沉迷未强制**：Gate 只是查询端点，未作为前置中间件挂在 advance-phase/下指令/建局；RecordPlaySeconds 零调用→防沉迷计时器永不累加。

### 跨玩家与 PvE（elite 已接，其余 ops-only）
- **单人 elite 遭遇全链路 + 前端入口已接主客户端**（done）。
- field_boss/world_boss/七种交互/consent 三档/社会客体撮合：**全部仅 ops-HTTP 可达、无玩家 UI、无自治驱动**（候选/队伍须外部手传）；region.threat_level 触发仍是硬编码常量。
- 跨玩家事件→收件箱桥接 `SurfaceCrossEventsForCharacter` **无 live 调用方**。

### 商业化剩余
- Google IAP token 源是静态 env（非 service-account OAuth2 自动刷新，注释自陈生产勿长用）。
- ListEntitlements 无 HTTP 路由（玩家查不到已购权益）。

### 前端 / 实时
- WS `llm_interaction` 单独事件前端无分支（改从快照读）；WS 心跳 120s 读超时永不刷新→周期性短暂断连后自动重连兜底。

## 六、完全没做（仅文档 / 仅 stub / 死代码）

### 账户级成本闭环（三连缺，互为前提）
- **session 未绑定 account_id**（types.go 无 AccountID）——账户级配额无从归集 LLM 成本的**根因**。
- **chargeLLMQuota / billing.AddSpend 零调用方**（死代码）→ account_llm_quota.spent 永远为 0。
- **账户级配额闸未接 LLM 路径**：CheckQuota 是孤立查询端点，不拦截任何 LLM 调用——PRD §3.1「付费成本上限」机制实际未生效。

### 引擎原语 / 战斗
- **status.Mutator.ApplyBatch 零非测试调用方**（死代码）——承诺的「每决策 ~15 次 DB 往返收敛」完全未兑现。
- reaction_queue 确定性决策解析器整套 Deprecated 死代码（仅提示词半边在用）。

### 大世界 / 跨玩家（设计承诺，原语在库未接线）
- 多实例真正分片的 **leader 选举 / region→实例稳定哈希 / rebalance**（当前只是「抢租即处理」对称争用切片）；region 注册表**无任何包外调用方**。
- 关系图传播 + 二手消息失真 + propagation_log + blood_feud 衍生撮合（§1.4/§2.7）整块未接线。
- regions PK 与 world_ticks 复合 PK 语义待对齐（region_id 全局唯一时不可触发，接线前需统一）。

### 命运 / 叙事产品化
- 可分享「死亡传记卡」（Spotify-Wrapped 竖图，变现/病毒核心物）；持久化编年史 chronicle_entries + 「回到那一刻」链接；角色创建「仇人/宿敌可追溯记忆种子」；翻译层 §4.1 模板；redline 离线宪章结构化数据源。

### 前端 UI 覆盖（后端能力悬空）
- **治理 UI**（举报/审计/隐私擦除）：零玩家界面。
- **商业化/实名 UI**（内购/实名/配额）：前端连 API wrapper 都没有——付费与防沉迷玩家侧不可达。
- **跨玩家/世界/consent/ops UI**：无玩家或运营前端。

### 合规前置门（PRD 列为可致项目归零）
- 年龄分级开关 / 未成年模式实际强制 / 防沉迷计时实际累加——能力在库但**未挂在任何玩法端点**，客户端不调即绕过。

### 测试
- 前端测试「写了不跑」（仅 tsc 编译检查，无 runner 实际执行抗命卡解析断言）；combat_shake 主循环插桩缺端到端 DB/LLM 集成测试；region 注册表 MySQL 路径仅 SQLite 测试覆盖。
- 漏斗：5/7 P1 常量补全但无实际 Emit 点（留存/转介/合规拦截漏斗无数据）。

## 七、建议优先级（更新版剩余）

### P0 — 让已建能力对玩家真正生效
- 命运卡**倒计时前端渲染**（扩 FateCard 类型 + 渲染剩余小时）——M2② 收尾。
- **主指挥客户端建局路径接 SeedVillage**（主玩法局兑现 20 人关系网，否则命运层无锚可点亮）。
- **未成年/防沉迷 Gate 挂成前置中间件** + RecordPlaySeconds 接入（否则合规门=没门）。
- field/world boss + 七交互 + consent 给**玩家 UI 入口**（后端已就绪，纯前端 + api wrapper 缺口）。

### P1 — 闭环与可靠性
- **session↔account 绑定** → 接 chargeLLMQuota 进 LLM 成本结算 + CheckQuota 进 LLM 前置拦截（打通账户级成本封顶，三连缺的根因优先）。
- 接 **status.ApplyBatch** 进决策落地（兑现 DB 往返降本）。
- 治理/商业化**前端 UI**（举报/审计/隐私 + 内购/实名/配额）。
- 补 5 个未 Emit 的漏斗埋点 + 前端测试 runner + combat_shake 端到端测试。

### P2 — 自治驱动与规模化
- PvE/跨玩家/撮合的**自治驱动**（region.threat_level 累积触发、世界自然产出社会客体再择人），替代 ops-HTTP 手动。
- 关系图传播 + blood_feud 衍生撮合（§1.4/§2.7）。
- 多实例真正分片（leader 选举 / region 稳定哈希）；Google IAP service-account OAuth2；regions PK 对齐。
