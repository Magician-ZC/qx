## 一、一句话总评

**项目已建成一台「机制扎实、原语精良」的单局引擎，但还不是一款「玩家可感知闭环」的产品。**

底层结算层（回合循环 / AI-LLM / 生存经济 / 社交关系 / 记忆叙事）已成体系、端到端可跑且多数接入主链路；工程质量最高的引擎原语层（仲裁 / 相关性 / 归因 / 调度 / 批写）全绿有测试。但凡涉及**世界级编排**（大世界 region 分片、跨玩家传播）、**前台产品化**（命运四槽 / PvE / 治理 / 抗命卡在主客户端的呈现）、**规模化与商业化**（水平扩展、付费、漏斗埋点）这三层，要么停在 default-off 灰度、要么只在文档里——这正是当前的短板带。

## 三、已完成（端到端可用、接入主链路）

### 核心回合循环（最成熟）
- 部署/执行两阶段状态机；执行→部署边界统一结算（怀孕/信鸽/饥饿/记忆衰减/敌方方针刷新）。
- ATB 真·行动条调度（gauge 累积 threshold=100、6 分项速度、同阵营连续抢占 ×0.85 势头惩罚）。
- 三类指令 doctrine/task/order + 指令力经济（默认 3/3、每部署回合 +2、order/严格 task 各扣 1）。
- 指令→LLM 归一化→上下文聚合→单位决策完整数据流；归因强制 + GateSurprise + Echo 默认线上开启。
- 服从度判定（风险≥0.9 才进概率、上限 0.85、FNV 确定性掷骰、五来源归因、四态 steady/concerned/reluctant/refused）。
- 异步执行（后台 goroutine + panic 恢复 + 45min 超时 + 进度回调 WS + 2min stale 自动重接 + best-effort 状态保存）。
- 断线重连阶段边界快照（`session_phase_snapshots` 双键、裁剪 160、历史时间轴）。
- 单人模式敌方全局方针 LLM 生成 + 启发式 fallback；执行主循环单位 AP 轮转。

### AI-LLM 子系统
- 统一 `GenerateJSON` 入口无旁路（~25 个 LLM 点全经此）；gojsonschema 强校验，不接受部分结果。
- 多 provider 故障转移 Primary(OpenAI)→Secondary(DeepSeek)→Tertiary(fallback)；同签名端点多 key 原子轮询（含 OPENROUTER_API_KEYS）。
- LLMInteraction 压缩留痕（最近 48 条、仅最近 8 条留完整 prompt，旁路表保全量供审计）；成本估算按 provider 单价、缓存命中计 $0。

### 社交关系外交服从
- 四轴关系 trust/fear/affection/rivalry 双层 clamp(±10)，广泛挂到战斗/援助/技能/信鸽/交易/情报/村庄织网。
- 抗命溯源卡 + 固定成长旁白（§3.3）；外交禁令策略从 NL 解析并强制；势力关系状态机 war/neutral/allied。
- 单人敌方 LLM 全局策略；情报线（卧底招募/泄密/资产暴露与周期回传，双方同意门控）。

### 生存经济生产（落地最名副其实）
- 饥饿结算（地形消耗/动作消耗/断粮惩罚/进食兜底）；确定性天气（FNV，影响移动/战斗/采集/事件）。
- 生产采集锻造强化穿戴、贸易（同意门控+关系联动）、互动对话、建筑（建造/续建/拆除/攻击设施）、随机事件（确定性分支+人格风险偏置）——全部进 `executeDecision` 统一分发、经候选装配喂 LLM。

### 记忆叙事（机制层最全）
- 结构化记忆（分类容量上限 + 指数衰减 tau=120 + importance^2.5 + permanent 永久 + 闪回 FTS 召回 + recall 强化 + memory2 长期压缩），每决策与回合边界真实驱动 LLM 上下文，记忆 ID 暴露进归因快照。

### 实时通道与前端

## 四、部分完成（有能力但未接主链路 / 默认 flag 关 / 半接入）

### 引擎原语「建好未接」断层
- **decision.Router 真短路默认关**：仅 `reflex_shadow` 影子遥测，未在玩家战斗主循环短路任何 LLM；「<2% 上 LLM」仅设计目标、无实测埋点。
- **status.Mutator.ApplyBatch 零非测试调用方**：批写能力建好测过等价，但 session 决策落地仍走单条 `Apply`（6 处），承诺的 DB 瓶颈收敛未生效。
- **scheduler / encounter / attachment 只接 region-runner**，而 region-runner 整体 default-OFF（叠 ENABLED/APPLY/THREATS/THREATS_APPLY 四 flag 才真打）；attachment 的 ReturnVisits 恒 0。

### 命运层玩家主链路断点
- **onboarding 不触发 SeedVillage**：`bootstrapCharacter` 不传 `with_village=1`，「身边二十个人」是预览文案而非真生成（§4.5 未兑现）。
- **抗命溯源卡前端零解析**：后端把 `source=..|phrase=..|narration=..` 编码进 Reasoning，FateView/FateApp 不解析、不渲染 refused reasoning，玩家看不到这张卡。
- **跨玩家事件只 Record 不 Surface**：`SurfaceCrossEventsForCharacter` 零生产调用，普通世界事件进收件箱仅 death/threat 两条路径。
- 持久相关性锚已建但 goal/redline 锚事件侧难点亮（半接入）；M4 牵挂仅喂威胁惩罚未驱动后果层解锁/UI；consent 路由全在 opsTokenGuard 后非玩家 UI 可达。

### AI-LLM 规模化降本未做
- 预算护栏仍是「每会话 $3 硬上限」；§11 设计的「每角色/每用户 BudgetGate + llm_budget_accounts/llm_charges 流水 + 分 tier 路由」未实现。
- prompt 缓存能力齐全但仅 unit_decision 一处 Cacheable；§7.3 验收门（system 前缀稳定化 + 命中率>60% + 计费 token 降>50%）未做，真实命中率大概率偏低。
- batch 去重原语完备但只接「单位身份叙事」一条非核心链路，执行主循环逐单位决策仍串行单发、未按 region 批量聚合。

### 跨玩家与 PvE（原语成、编排未成）
- elite/field_boss/world_boss 三档都有真实 session 结算链路（多回合 combat_roll → AllocateLoot 分赃 / DegradePenalty 后果闸 → Mutator 留痕 → 祖魂卡）；world_boss 还实现共享血池+频率无关贡献账本+arbitration 排他件（反 P2W 核心）。
- **但只有 elite 经 region-runner 主循环 StrategicFork 自动触发**；field_boss/world_boss/七种交互/撮合/consent **全部只能 ops-token HTTP 手动触发、无后台世界驱动、前端零集成**。
- 撮合四因子由调用方传入（无世界态自动算候选）；consent 的 requires_consent 仍由 bool 决定（未经 decision/attribution 自治）；ExpireStaleConsents 无后台定时调用。

### 持久化大世界（MVP 单实例可跑）
- region-runner 离线发动机骨架→HOT-LLM→PvE 接入→单位 seed→乐观并发硬化**全链路打通且经对抗评审修了多个 load-bearing**，但默认关、灰度未开；离线动机栈只做 L1+L3 反射四动作，L2 reassessGoal / 冻结清单上交命运收件箱未做。
- worlds/regions/world_ticks **只建了 worlds**（region==sessionID 隐式约定，tick 由真实 Unix 秒派生）；保留期清理已接，但单位死亡未主动清唤醒队列/在途作业（留孤儿 wake 风险）。

### 治理安全 / 战斗 / 前端
- 举报只写不审（Resolved 字段从无代码置位，无 triage 闭环）；content_safety 已接 3 chokepoint 但本体小词表、默认关、extendedModeration 仍 no-op；反 P2W 红线仅接 3 条专用路径未自动触发执行主循环。
- 反应队列被作者主动 Deprecated（仅给 LLM 喂提示信号，规则版不接主链路）；combat_roll 缺 GDD 点名的直接单测。
- WS `llm_interaction` 事件前端无分支被静默丢弃（改从快照读，功能不缺但通道冗余）；Unciv UI 五项基本落地，缺「下一个闲置单位」跳转；命运四槽/抗命卡/社交关系/治理 UI 严重未覆盖后端能力。

## 六、跨子系统主线缺口（横切综合）

1. **「原语已实现但未接主链路 / default-off 灰度」——最普遍的成熟度断层。** 涉及引擎原语（Router 真短路、ApplyBatch 零调用、scheduler/encounter/attachment 全挂 region-runner）、跨玩家与 PvE（除 elite 外全靠 ops-HTTP 手动）、大世界（四 flag 叠加才生效）、战斗（combat_shake 死代码）。**这是把引擎变成游戏的最后一公里，不补则前期投入无法兑现为体验。**
2. **前台 UI 严重未覆盖后端能力——两套割裂的前端。** 命运四槽只活在独立 `#fate` 路由，主指挥客户端 App.tsx 完全不接命运/PvE/治理；elite-encounter 有 endpoint 前端零调用；举报/审计/隐私 API 就绪却无界面入口；抗命溯源卡前端零解析。**本作差异化卖点「命运感/牵挂/抗命」主战棋玩家感知不到，削弱核心假设的可验证性。**
3. **大世界「世界化」与水平扩展未落地——规模化地基缺失。** 仍是单局双阵营轮流；regions/world_ticks 表缺失、region==sessionID 隐式约定；单进程单 runner 无区锁/多实例；连接池未扩容。**「承载无限 agent 的活世界」核心愿景的吞吐地基缺位。**
4. **商业化与验证数据管道几乎为零——项目能否成立的门无数据支撑。** 付费/配额/ARPU 全无；漏斗埋点半成品。**「留存×成本×变现三门」中变现门、留存门无数据可判，验证实验无法得出 go/no-go 结论。**
5. **合规前置门完全缺失——可致项目归零的硬门。** 年龄分级/未成年模式/实名/防沉迷零实现；高危 ops 端点漏鉴权。**上线前的法务/越权硬门，不补则无法对外。**
6. **确定性结算层测试缺口集中。** 回合循环/记忆衰减/combat_roll 这些「结算用代码」本应可测的确定性逻辑回归全靠人工，是规模化前的可靠性盲区。
7. **「离线宪章 / redline / 翻译模板 / 记忆种子」系列承诺未落地，削弱归因可追溯链。** 红线/记忆种子缺位让「突然复仇/叛变」高戏剧性事件失去可追溯前因，弱化设计宪法核心约束。

## 七、建议优先级

### P0 — 打通可玩闭环（让玩家在默认配置下感知到核心差异化）
- 把**命运四槽 + 抗命溯源卡**接进主指挥客户端 App.tsx（后端已就绪，纯前端缺口）——核心卖点唯一可感知出口。
- onboarding 接 `with_village=1` 触发 SeedVillage，兑现「身边二十个人」，否则命运层无关系锚可点亮。
- 把 **combat_shake** 接进 generateUnitDecision，或确认弃用并清死码——避免已建战斗深度白费。
- 在 region-runner 默认关前提下，至少让 **elite/PvE 遭遇**有一条玩家可在主客户端发起/看见的路径。

### P1 — 体验完整 + 可靠性兜底
- 补 GDD 点名的**确定性结算单测**（ATB 等价基线、combat_roll、记忆衰减/cap），把回归从人工转为基线护栏。
- 落地命运 **M2/M3 硬规则**（每日≤3 待决策、过期倒计时兜底、D0-D3 后果分级闸通用化）——防轰炸与不可逆后果是命运体验的承诺底线。
- 接 **status.ApplyBatch** 进决策落地；修 **trade.go 死代码**与 **hunger.go 绕过 Mutator**两处技术债——清掉污染 CI 的历史告警与不变量违例。
- 补全 **analytics 漏斗埋点**（account/character/charter/inbox_opened/share 等），让北极星 D2 处理率与留存分层有真实数据源。

### P2 — 规模化与商业化（验证通过后再投）
- **region 分片 + 多实例 worker 池 + 连接池扩容**（MySQL→64-128），兑现大世界水平扩展地基。
- 落地**商业化层**（天命配额节流 + 月卡 SKU + IAP 收据校验 + 付费/ARPU 埋点）与成本仪表盘红黄告警，打通变现门。
- 实现**合规前置门**（年龄分级/未成年模式/实名/防沉迷）并给高危 ops 端点补 opsTokenGuard——上线硬门，但需在确认产品方向值得上线后再投入。
