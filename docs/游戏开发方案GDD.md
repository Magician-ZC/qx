## 1. 一句话与高概念

**《群像》是一款 AI 命运养成沙盘**：你捏一个「人格种子」丢进一个永不停止的共享世界，离线时它由 AI 自主地谋生、结盟、相爱、复仇、背叛；上线时你回来开盒，看它做出了你没料到但合理的选择，再微调种子让它长成你想要（或更出乎你意料）的人。它做的每一步都被忠实写进不可篡改的编年史。

**一句话定位**：CK3/BitLife 的「命运感」× AI agent 的「活世界」× 电子宠物的「牵挂」。

## 2. 核心玩具（Core Toy）—— 必须先好玩

> 这是评审揭示的最大盲区修正：原方案把「读编年史/处理收件箱」误当 core toy，而那只是「通知整理」。真正的玩具是下面这个**开盒循环**。

**Core Toy =「人格炼蛊」**：
```
捏种子(野心/伤痕/红线) → 投放进世界 → 回来开盒看它做了什么我没料到但合理的选择 → 微调种子 → 再开盒
```
这是**养蛊 + 盲盒 + 电子宠物**的合体。玩家反复想做、做了就爽的最小动作 = **「看我捏的种子长出了我没指定的行为，并为之惊讶」**。

**好玩假设（must-test，见验证设计 + 本文 §8）**：玩家能从「一个符合人格、但我没指定的合理意外」中获得持续的开盒快感。这是整个产品的乐趣地基，必须最先、最便宜地验证（不是验「会不会回来」，是验「玩得爽不爽」）。

一旦把它定为 core toy，几处设计自动改写：
- 捏人**不是漏斗障碍，是第一个玩具**（见 §4）。
- 「微调种子」是玩家**主动追求、看得见效果**的玩法环，而非被动副作用（见 §5 共鸣度）。
- 乐趣度量从「收件箱处理率」改为「**惊喜命中率**」（见 §8）。

## 3. Moment-to-Moment：一次典型会话逐步拆解

| 秒 | 玩家动作 | 看到的反馈 | 设计意图 |
|---|---|---|---|
| 0–10 | 打开 App（被 D2 推送或主动） | **状态卡**一行：「离线 11h｜阿采还活着 HP64 在北岭林地｜2 件待决策 1 紧急」 | 60 秒回归，先知道「它怎么样了」 |
| 10–40 | 滑动 1–3 张**高光卡** | 每张一句≤30 字旁白+情绪色；轻交互〔意料之中/有点意外但合理/太离谱〕 | **开盒**——core toy 的核心爽点；同时采集惊喜命中率 |
| 40–120 | 打开**收件箱**，处理 1 条悬念决策 | Copilot 给 2 选项+角色倾向+后果预测；选/改写；**处理回执**「阿采更信你一点了」 | 「介入命运」+ 掌控可感（共鸣度 +） |
| 120–150 | （可选）高光节点**直控**几回合 | 决斗/告白/复仇的实时操作，事后标进编年史 | **唯一有手感的那一下**（见 §5），最强分享素材 |
| 150–180 | 会话**收尾卡** | 「这次你为它做了 3 个决定，因为你它[守住了断刃]，下次它会面对[预告]」 | 结局感 + 影响累积可见 + 埋下次 D2 钩子 |

**关键**：原方案此流程全是异步消费（读卡→点按钮→关 App），缺「实时有手感的那一下」。§5 通过「直控解锁化」把它补回来。

## 4. 首日 30 秒啊哈：即时人格快照（Instant Soul Preview）

**不要让玩家创建完就下线等明天**——那把「第一次啊哈」全押在 D2，而 D1→D2 情感账户为 0。

**改法**：创角后立刻用 1 次（可缓存/可降级）LLM 对种子做一段 15 秒压缩快进：
> 「你刚把阿采放进世界。看看它独处的第一个小时——」
> 〔高光卡〕「它绕开了村口的人群，独自在桥下扎营，对一只受伤的鸟说了第一句话。」

内容必须是「**符合它人格（游侠+社交↓）但玩家没指定**」的微选择。这 30 秒就是首次啊哈：玩家第一次看到「我捏的种子真的长出了行为」。代价仅 1 次 LLM，却在 D0 就把「情感账户余额为 0」解决一半。**A/B：有快照组 vs 无快照组的 D1→D2。**

**捏人即玩具**：每个种子维度（野心/伤痕/红线）给一句「它大概会这样活」的即时 LLM 预览，让玩家在捏的当下就体验「我的选择会塑造一个人」。随机模版一键即走（展示缝合好的角色卡：一句 biography+一句欲望+一句伤痕），自捏路径最多 3 步每步只问 1 件事。

## 5. 自治 vs 掌控：玩法层解法（差异化命门）

原方案只在「验证层/护栏层」处理这对张力（一致性旋钮、违意 A/B）。这里给**玩法层**的正面解法——让玩家**通过塑造获得正向掌控成就感**，而不是靠限流去压。

### 5.1 共鸣度 Resonance（园丁式掌控）
新增一个玩家可感的进度指标 **共鸣度** = 玩家近期决策/嘱咐 与 角色自发选择 的契合度滑动窗口（是 §5.7 的 Loyalty/性格漂移三轴的**玩家可见聚合视图**）。
- 玩家通过「追认它的选择 / 给它讲我的价值观 / 在岔路口选与它倾向一致的项」**提升共鸣**。
- 共鸣高 → 角色岔路口更可预期（但仍保留人格惊喜）；共鸣低 → 它更我行我素。
- **掌控不再 = 直控压制 LLM，而 = 「我把它养成了懂我的人」的园丁成就感**——正面化解张力。

### 5.2 直控松绑：从「要收税的违规」改成「养成的奖励」
原 §5.5 把直控设计成消耗指挥专注+每日限时+「防架空卖点」，等于把唯一有手感的玩法关进小黑屋。**改为**：
- 直控时长/权限**随共鸣度/牵挂解锁**——你越懂它、它越信你，你越能在关键时刻「附身」它。直控成为**养成的奖励出口**，不是要堵的漏洞。
- 乐趣从「手操战棋」重定向到「**高光时刻的共同演出**」：只在它人生关键节点（决斗/告白/复仇）接管几回合，事后这段被特别标进编年史 =「**我亲手打的那一仗**」——既保留实时手感，又不架空 LLM（日常仍自治），还制造最强分享/禀赋素材。

## 6. 世界与角色系统（概要，详见设计方案 §2–§6）

- **世界**：分 Region 不分服（同一份权威世界）；1 tick=真实 6 分钟；Hot/Warm/Cold 三级活跃度承载无限 agent；**赛季制**（每季从神话层+burn-in 重开、归档名人堂——同时解决公平起跑/运营节奏/技术降负/冷启动密度）。
- **角色创建**：指定模版（6–8 原型）/ 随机模版（确定性抽签+LLM 缝合）；新增六维**野心向量** + **离线宪章**（红线）。
- **自由发展**：三层动机栈（L1 生理护栏 / L2 LLM 自评目标 / L3 野心向量）→ 谋生/结盟/复仇/叛变自发涌现。
- **离线自治 + 四模式接管**：指令 / 直控（已重做为奖励，§5.2）/ Copilot / 复盘；上线 60 秒「状态卡+高光卡+收件箱」。
- **留痕→编年史**：events 派生四层叙事，**以可一瞥的高光卡/可分享图为主、长文懒渲染**（§11.4）；LLM 只渲染不编造（幻觉回扫 0 容忍）；不可靠多视角=战争迷雾式历史。
- **生命循环**：衰老/死亡/**血脉传承**（核心留存钩子，死亡传记卡=最强分享触发点）。

## 7. 引擎架构：决策用 LLM，结算用代码（含已落地代码）

### 7.1 核心架构原则（设计方案 §1.5）
**LLM 决定「做什么」（意图），代码决定「结果如何」（结算）。** 三层执行模型：
- **① 反射层 Reflex**（纯代码，0 LLM）：绝大多数日常 tick。
- **② 决断层 Deliberation**（LLM，稀疏）：仅关键节点产出意图。
- **③ 结算层 Resolution**（纯代码 + FNV，0 LLM）：战斗/移动/经济/状态变更/零和裁决全部确定性算完，绝不逐步问 LLM。

这条边界一箭三雕：**成本**（LLM 从「每步」收敛到「每决策点一次」）、**公平**（结算确定性→付费改不了已开打的战斗结果）、**可靠**（可复现、可审计、不幻觉）。

### 7.2 已落地的工程升级（本轮新增，`backend/internal/engine/`，零破坏 + 仓库首套测试）

| 包 / 方法 | 作用 | 对应设计 | 测试 |
|---|---|---|---|
| `engine/decision`·`Router.Route` | 决策层路由：安全反射优先→关键节点升级 LLM→日常零 LLM。把「<2% 上 LLM」从降级模式扶正为常态 | §1.5 三层模型·决策层 | 8 用例✅ |
| `engine/arbitration`·`Resolve` | 零和仲裁原语：胜负仅由 Score+确定性掷骰，**与频率/顺序无关**（反 P2W）。A-Res 加权抽样保证胜率∝Score | §11.3 仲裁 tick | 6 用例✅（含一处真实公平 bug 被测试当场捕获修复） |
| `status.Mutator.ApplyBatch` | 批量状态写入：每决策 ~15 次 DB 往返 → 按单位读一次/写一次 + 单事务批量插事件，语义等价 | §11.2 引擎瓶颈 | 5 用例✅ |

验证：`go build ./... && go vet ./internal/engine/... && go test ./... && go run ./cmd/statuslint ./internal/engine/...` 全绿。

```
运行方式（backend/ 目录）：
  go test ./internal/engine/...      # 运行新引擎测试
  go run ./cmd/statuslint ./...      # 状态字段直写守门
```

### 7.3 引擎升级技术规格（后续，O9 剩余项）
评审指出三处原方案过乐观，须按下表真实分级，配验收门与回滚锚点：

| 改动 | 原方案口径 | 真实分级 | 正确改法 |
|---|---|---|---|
| events 作用域迁移 | 「换 SessionID=零改动」 | **高改动面**（最热写路径、无事务） | **加列不改义双键过渡**：events 加 `world_id/region_id/tick`（可空）保留 `session_id`；Mutator 接 `ScopeContext` 同写新旧键；回滚=停读新列 |
| prompt 缓存 | 「立即可做 $11/天」 | **需新工程项**（仓库无此功能、各 provider 语义不一） | **前缀稳定化重构**优先：~2800 token 静态指令提到 system 最前段且字节固定（缓存按最长公共前缀命中）；验收门=命中率>60% 且单决策输入计费 token 降>50% |
| 收件箱 | 「不是新表」 | **需开不变量旁路** | `events.EmitProcessEvent`：纯流程事件（Field/Delta 空、Category=player_decision）与 Mutator 共用同表同事务，不过受保护字段 clamp |

**先写的 5 个测试**：①Mutator 批写前后 events 序列与最终态等价 golden（✅ 已写）；②combat_roll FNV 在 region/tick 作用域下确定性；③ReconcileState 拆表对账；④ATB 跨 tick gauge 调度与单回合全序排序的语义等价基线；⑤region 租约锁并发抢占。**每阶段「可玩性回归基线」**：固定 seed+固定指令序列→固定 events 流，重构后语义级回放一致才允许合并。

## 8. 乐趣度量（进 MVP 看板，与留存并列）

| 指标 | 口径 | 作用 |
|---|---|---|
| **惊喜命中率** | 高光卡轻交互「有点意外但合理」/ 全部卡 | 核心乐趣直接代理（高=养蛊成功=留存） |
| **OOC 率** | 「太离谱了」/ 全部卡 | 负体验看门狗（高→收紧一致性旋钮/改进解释生成） |
| **乐趣自评** | 可玩测试后「下次还想玩吗/最爽的一下/最无聊的一下」 | 证「好玩」而非只「回来了」 |

> 在验证 ② 与 ③ 之间插入 **5–8 人面对面可玩测试**（捏种子→看即时快照→处理一个岔路口→看会话收尾卡，出声思考+乐趣自评）——这是发现「留存假阳性（②靠人肉高质量惊喜成立、③真 LLM 给不出）」的最便宜手段。

## 9. Vertical Slice 规格（融资交付物）

**定义**：1 角色×1 区域×1 条完整情感弧×**发行级质感**的 30–40 分钟体验。**VS ≠ 验证 MVP**（MVP 证机制可丑，VS 证质感必须美）。

**必含**：
- 发行级 UI 全流程：创角→即时人格快照→设嘱咐→下线→D2 推送→上线第一分钟收件箱。
- ≥3 张手工打磨的高光卡视觉模板 + 情绪色系统（紧张红/温情橙/危机紫/暧昧粉/平静蓝）。
- 1 张可分享的死亡/高光**传记卡**（Spotify-Wrapped 式竖图，变现+病毒核心物）。
- 1 条精心设计的「违背意志」母题事件 + 共鸣度可视化。
- **真实 LLM 驱动**（非人肉）。

**Exit criteria**：投资人/发行自己玩 30 分钟后说出「**我牵挂这个角色**」。

**投入**：4–5 人×2–3 月、约 ¥80–150 万，作为「验证 GO 后的第一个融资里程碑」。

## 10. 内容生产管线（产能量化）

把「LLM 生成」拆成「**人写骨架 + LLM 填充**」，骨架是需持续投入的人力内容：

| 资产 | 内容 | 产能假设 |
|---|---|---|
| **决策事件母题库** | 上线≥40 个情境母题（战书/求婚/复仇/背叛/天灾/夺权…），每个含触发条件+两难结构+Copilot 选项骨架+过期兜底文案 | 1 名系统/叙事 ≈2–3 个/周 |
| 高光卡视觉模板 | 情绪色×事件类型矩阵 | 美术，按矩阵铺 |
| 模版原型 | 每赛季新增 1–2 个 | 叙事 |
| 推送文案库 | 5 变体→按情境分支 | 叙事 |
| 冷启动内容 | 神话层+burn-in 活史（开服一次性大投入） | 叙事+系统 |

**赛季常态内容人力 = 叙事 1.5 + 美术 1 + 本地化 0.5**。这一章回答发行最常问的「你们靠什么持续给玩家新东西」。

双向审核管线（输入侧 prompt-injection/越狱检测、输出侧 NSFW/暴力/仇恨分类器，命中回退规则模板）+ 年龄分级与未成年人模式（关闭恋爱生育/降暴力）+ ToS 中的「**角色行为下限契约**」（平台保证 AI 永不让你的角色做某类绝对红线行为）+ LLM 安全红队作为发行前 gate。

## 12. 设计风险登记（玩法侧）

| 风险 | 缓解 |
|---|---|
| **moment-to-moment 是「通知整理」非「玩游戏」**，留存全押在最未验证的「牵挂」 | core toy 显式定义为人格炼蛊（§2）；D0 即时快照在牵挂建立前就给乐趣（§4）；低牵挂期(D0–D3)靠捏人惊喜/共鸣度起步/可分享开盒兜底 |
| 直控被设计成被压制的反模式，阉割了唯一手感 | §5.2 直控改为养成奖励 + 高光共同演出 |
| **好玩假阳性**：②人肉高质量惊喜成立、③真 LLM 给不出同等惊喜密度 | ②加真 LLM 盲测臂（验证设计 O5）；②↔③间加可玩测试；惊喜命中率/OOC 率上看板（§8）；真 LLM 惊喜质量是 ③ 的一等监控 |
| 自治太强 → 新玩家觉得「不是我的角色」 | 共鸣度园丁玩法给正向掌控（§5.1）+ 一致性旋钮（设计 §11.5） |
