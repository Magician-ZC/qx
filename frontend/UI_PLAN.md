# 群像前端 UI 改造清单与计划

## 1. 顶部状态栏 (Top Bar) 改造
*   **当前状态**：使用 `map-hud` 悬浮药丸（Pill）展示 Live、回合数 (T1)、阶段等。
*   **修改计划**：
    *   将这些零散的药丸整合成一个**横跨屏幕顶部的深色半透明状态栏**。
    *   **左侧**：添加汉堡菜单图标（占位）和当前阵营/玩家标识。
    *   **中侧**：显示回合数、当前游戏阶段（Phase），并预留资源数据（金币、科技等，如果没有真实数据则使用现有数据或占位符）的位置。
    *   **右侧**：放置类似“概览”和“状态（Outcome）”的按钮/信息。

## 2. 左下角选中目标与操作区 (Selection Info & Actions Panel)
*   **当前状态**：可能通过统一的 `floating-toolbar` 或右侧卡片展示。
*   **修改计划**：
    *   在**屏幕左下角**创建一个固定的信息面板。
    *   **交互逻辑更新 (Unciv 风格)**：
        *   每个地块只能有一个单位（目前看后端/渲染逻辑可能已经支持，需要确认 UI 交互）。
        *   点击地块时：
            *   如果有单位，面板优先展示单位信息（头像、名称、血量 HP、经验 XP 等），附着单位的指令按钮（“驻守”、“探索”等）。
            *   并且提供一个按钮可以**“点开展示详情”**。
            *   如果没有单位，面板展示该**地块属性**（地形类型、产出、防御加成等）。

## 3. 右侧/右下角操作区 (Quick Actions & View Controls)
*   **修改计划**：
    *   在右下角添加大型的“下一个闲置单位”或“结束回合”主按钮。
    *   将工具面板切换按钮（编队、LLM记录等）移动到右侧边缘或右下角垂直排列，采用圆形图标。

## 4. 地块渲染优化 (Tile Optimization)
*   **修改计划**：
    *   参考 Unciv，地形的渲染应该更清晰地区分地块边界和地形特征。
    *   选中地块时，增加更明显的六边形高亮边框。
    *   点击地块时触发 UI 联动，不仅仅是 `selectedUnitID`，可能需要引入 `selectedTile` 概念来展示地块信息。

## 5. 整体视觉风格与配色调整 (UI Theming)
*   **修改计划**：
    *   修改 `styles.css`，弃用白底大圆角卡片。
    *   采用**游戏 UI 风格**：深蓝色/灰黑色半透明背景、直角或极小圆角、白色主文本。
